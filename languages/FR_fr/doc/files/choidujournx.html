Utilisation de ChoiDuJourNX

 

Dans ce tuto nous allons voir comment utiliser le homebrew ChoiDuJourNX pour pouvoir mettre à jour la Switch, que ce soit l'emunand de SX OS ou la sysnand (la console en elle-même). Dans ce tuto, je partirai du principe que vous savez lancer un CFW et un homebrew, se référer à cette FAQ si vous ne savez pas comment faire. Je partirai également du principe que vous avez préparé votre SD à l'aide de mon script permettant de mettre en place les données de mise à jour ainsi que ChoiDuJourNX sur la SD

Attention: Ne pas mettre à jour la sysnand d'une consol patchée via cette méthode et surtout, ne pas utiliser ChoiDuJour-NX sur une console équipée d'une puce SX Core ou SX Lite. Notez également que ce homebrew n'est plus compatible avec les dernières versions d'Atmosphere, utilisez Daybreak à la place qui fonctionne plus ou moins de la même façon.

Avant de commencer, notez que l'installation d'un firmware via ChoiDuJourNX activera automatiquement l'auto-RCM si une option n'est pas décochée pendant la procédure d'installation (lire le tuto jusqu'au bout pour les choses à savoir si désactivation de l'auto-RCM), que la mise à jour concerne l'emunand ou la sysnand cela n'y changera rien. Notez également que l'auto-RCM pourra, si vous le souhaitez, être désactivé juste après les manipulations, voir la fin de ce petit tuto pour en savoir plus. Notez qu'en cas d'une mise à jour de l'emunand uniquement, l'auto-RCM peut être désactivé sans risque si la sysnand n'a pas été mise à jour.

En premier lieu, il vaut mieux s'assurer que la console ait de la batterie, 50% histoire d'être certains que la console ne s'éteigne pas en plein milieu du processus.

On lance le homebrew ChoiDuJourNX, on clique sur le dossier dans lequel se trouve le firmware (via mon script se sera la mention "fw_" suivi du numéro du firmware). On se retrouve dans un dossier contenant une multitude d'éléments nommés en hexadécimal et portant l'extension ".nca", c'est le bon endroit donc on clique sur "choose" en bas à droite de l'écran.

Le logiciel va proposer soit la version classique du firmware ou soit la version EXFAT. Je conseil d'installer la version EXFAT, même si vous n'utilisez pas ce format pour votre carte SD cela pourra toujours servir. Une fois choisi, une préparation se lance.

Une fois le premier chargement terminé, on clique sur "select firmware" puis une autre phase de chargement démarre.

Une fois la seconde étape terminée, on peut décocher la case "auto-RCM" dans le menu de gauche si on ne souhaite pas activer l'auto-RCM puis on peut cliquer sur "Start installation" et une fois que cela sera terminé, on clique sur "Reboot".

Si vous le souhaitez, l'auto-RCM qui a été activé lors de cette mise à jour (sauf si l'option a été décochée auquel cas il ne sera pas activé) peut être désactivé (voir la FAQ au sujet des Efuses et de l'auto-RCM pour comprendre les conséquences de tels choix) soit via Hekate via le menu "tools" ou soit via SX Loader via le menu de démarrage (je rappelle que pour y accéder il faut maintenir la touche "volume +" de la console jusqu'à l'apparition des options sur le menu) dans le menu "Options" puis "AutoRCM" puis "Uninstall" et enfin "Continue".
