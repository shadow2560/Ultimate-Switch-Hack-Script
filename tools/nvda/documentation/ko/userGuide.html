1. 머리말

NVDA에 오신것을 환영합니다!

NVDA(NonVisual Desktop Access)는 마이크로소프트 윈도우용 무료 오픈소스 스크린 리더입니다. NVDA는 음성과 점자를 통해 정보를 제공하기 때문에 시각장애인들이 정안인들보다 더 많은 비용 지출 없이도 윈도우가 동작하는 컴퓨터를 사용할 수 있게 합니다. NVDA는 NV Access와 커뮤니티의 후원을 받아 개발되고 있습니다.

1.1. 일반적인 특징

시각장애인들은 NVDA를 통해 윈도우 운영체제 및 그 밖의 다양한 응용 프로그램들을 사용할 수 있습니다.

주요 특징:

  • 웹 브라우져, 이메일 클라이언트, 인터넷 채팅 프로그램 및 오피스 관련 프로그램 등을 포함한 다양한 응용 프로그램 사용 가능
  • 80개국 언어 이상의 보이스를 내장한 음성엔진 탑제
  • 가능한 경우 글꼴 이름과 크기, 스타일, 맞춤법 오류와 같은 서식 정보 알림
  • 마우스가 위치한 곳의 글자와 들을 수 있는 부가적인 지시를 자동으로 읽기
  • 다양한 점자 디스플레이 지원 및 자동 인식 가능(디스플레이가 점자 키보드를 탑제한 경우 텍스트 입력도 가능)
  • 컴퓨터에 설치하지 않아도 USB 드라이브나 다른 휴대용 미디어에서 NVDA를 완벽하게 실행 가능
  • 손쉽게 사용할 수 있는 말하는 설치 과정
  • 54개국 언어로 번역됨
  • 최신 윈도우 (32 비트 및 64 비트) 버전 지원
  • 윈도우 로그온과 다른 보안 스크린에서도 실행 가능
  • 터치 명령을 사용하여 컨트롤 및 텍스트 읽기 가능
  • 일반적인 접근성 인터페이스 지원 예: Microsoft Active Accessibility, Java Access Bridge, IAccessible2, UI 오토메이션
  • 윈도우 명령 프롬프트와 콘솔 애플리케이션 지원
  • 시스템 포커스 하이라이트 기능

1.2. 국제화

중요한 것은 세계 어느 곳에 사는 사람이라도 사용하는 언어와 관계없이 기술에 대한 접근이 평등해야 한다는 것입니다. 현재 NVDA는 한국어와 영어 외에 다음 언어들로 번역되어 출시되고 있습니다: 갈리시아어, 그리스어, 그루지아어, 네덜란드어, 네팔어, 노르웨이어, 덴마크어, 독일어(독일 및 스위스), 러시아어, 루마니아어, 리투아니아어, 마세도니아어, 몽골어, 버마어, 베트남어, 불가리아어, 세르비아어, 슬로바키아어, 슬로베니아어, 스웨덴어, 스페인어(스페인 및 콜롬비아), 아이슬란드어, 아라곤어, 아라비아어, 알바니아어, 아일랜드어, 아프리카어, 에디오피아어, 우크라이나어, 이탈리아어, 일본어, 중국어 (간체 및 번체), 체코어, 카탈로니아어, 칸나다어, 크로아티아어, 키르기즈스탄어, 타밀어, 태국어, 터키어, 파르시어, 펀자브어, 폴란드어, 포루투갈어 (브라질 및 포루투갈), 프랑스어, 필란드어, 헝가리어, 헤브라이어, 힌디어 등 54개 언어를 지원합니다.

1.3. 음성 엔진 지원

NVDA 프로그램 메세지와 인터페이스를 여러 언어로 제공하는 것 외에, NVDA는 사용자들이 가지고 있는 음성 엔진에 따라서 컨텐츠를 어떠한 언어로도 읽을 수 있게 합니다.

NVDA는 무료 오픈소스 다국어 음성 엔진인 eSpeak NG를 내장하고 있습니다.

eSpeak NG 이외에 NVDA가 지원하는 음성 엔진에 대한 정보를 보시려면 지원하는 음성 엔진 섹션을 참고하세요.

1.4. 점자 디스플레이 지원

점자 사용자라면 점자 디스플레이를 통해 NvDA가 제공하는 정보를 읽을 수 있습니다. 그리고 점자 키보드를 사용하여 정자 및 약자 입력도 가능합니다. 또한 NVDA는 수많은 디스플레이 모델을 자동으로 인식하여 연결할 수 있습니다. NVDA가 지원하는 점자 디스플레이에 대한 정보는 지원하는 점자 디스플레이 섹션을 참조하세요.

NVDA는 다국어 점자 정자, 약자 및 컴퓨터 점자 코드를 지원합니다.

3. NVDA 다운로드 및 설치

아직 NVDA를 갖고 있지 않으시면 NV Access 웹사이트에서 다운 받으실 수 있습니다.

다운로드 섹션으로 가시면 NVDA 최신 버전 다운로드 링크를 찾을 수 있습니다.

다운로드 받은 파일을 실행하면 NVDA가 임시 버전으로 시작됩니다. 그리고 NVDA를 설치할 것인지 휴대용 버전을 만들 것인지 또는 단지 임시 버전만을 사용할 것인지에 대해 묻습니다.

만약 이 컴퓨터에서 항상 NVDA를 사용하시려면 'NVDA 설치'를 권장합니다. NVDA를 설치하면 부가적인 기능을 사용할 수 있습니다. 예를 들면 로그온 이후에 자동으로 NVDA 시작하기, 윈도우즈 로그온 화면과 윈도우즈 보안 화면 읽기 가능(휴대용 버전이나 임시 버전에서는 불가능합니다), 시작메뉴 및 바탕화면 아이콘 생성 등입니다. 설치 버전은 언제든지 단독으로 휴대용 버전을 만들 수도 있습니다.

만약 NVDA를 USB 드라이브나 다른 쓰기 가능한 미디어에 저장하여 휴대하고 싶다면 '휴대용 버전 만들기'를 선택해야 합니다. 휴대용 버전 역시 어느 컴퓨터에서도 언제든지 단독으로 NVDA를 설치할 수 있는 기능을 제공합니다. 그러나 NVDA를 CD 같은 읽기전용 미디어에 복사하고자 한다면, 다운로드 패키지만을 복사해야 합니다. 읽기전용 미디어에서 포터블 버전을 곧바로 실행하는 것은 지원하지 않습니다.

NVDA를 임시 버전으로 사용하는 것도 한가지 선택사항이긴 합니다. (예를 들면 전시를 목적으로 할 때) 그러나 이 방식으로 NVDA를 시작하려 한다면 매번 NVDA를 실행할 때마다 매우 긴 시간이 소비될 것입니다.

3.1. 휴대용 및 임시 버전 사용시 제한 사항

윈도우 로그인때 또는 로그인 후 자동으로 시작 불가 외에 휴대용 및 임시 버전은 다음 기능들이 제한됩니다:

  • 관리자 권한으로 실행된 프로그램 사용 불가 (NVDA 자체가 관리자 권한으로 실행된 경우는 제외; 관리자 권한으로 NvDA를 실행하는것은 권장하지 않습니다).
  • 특정 프로그램을 관리자 권한으로 실행하고자 할때 나타나는 사용자 개정 컨트롤 (UAC) 대화상자 접근 불가.
  • 윈도우 8 이상 사용시 터치스크린 조작 불가.
  • 윈도우 8 이상에서 윈도우 스토어 (Windows Store) 앱에서 브라우즈 모드 및 입력된 글자 출력과 같은 기능 사용 불가.
  • 윈도우 8 이상 사용시 오디오 도킹 사용 불가.

3.2. NVDA 설치하기

만약 다운로드 패키지에서 곧바로 NVDA를 설치하려면, 'NVDA 설치' 버튼을 누르세요. 만약 이 대화상자를 이미 닫았거나 혹은 휴대용 버전에서 NVDA를 설치하려면 NVDA 메뉴 - 도구 - NVDA 설치 메뉴 항목을 선택하세요.

설치 대화상자에서 NVDA 설치 및 기존 설치 버전을 업데이트할 것인지를 확인할 수 있습니다. 'Continue(계속)' 버튼을 누르면 NVDA 설치가 시작됩니다. 이 대화상자에는 차후에 설명될 몇가지 옵션이 있습니다. 설치가 완료되면 설치가 성공했습니다'라는 메시지가 나타납니다. 여기에서 'OK' (확인) 버튼을 누르면 새롭게 설치된 NVDA 설치 버전이 실행됩니다.

3.2.1. 호환되지 않는 추가 기능 경고

추가 기능 사용시 호환되지 않는 추가 기능이 있다면 이 추가 기능들이 비활성화될 수 있다는 경고 메시지가 나타납니다. 계속 설치하려면 호환되지 않는 추가 기능은 비활성될 수 있다는 체크박스를 체크하고 계속 버튼을 누릅니다. 또한 비활성될 추가 기능 목록을 볼 수 있는 버튼도 나타납니다. 자세한 내용은 호환되지 않는 추가 기능 대화상자 섹션을 참고하세요.

3.2.2. 로그인 중에 NVDA 사용

이 옵션을 선택하면 윈도우 로그온 화면에서 패스워드를 입력하기 전에 NVDA가 자동으로 실행됩니다. 또한 이 옵션은 사용자 계정 컨트롤(UAC)와 다른 보안 화면에서도 적용됩니다. 새로 설치하였다면 본 옵션은 기본적으로 선택되어 있습니다.

3.2.3. 바탕화면 바로가기 아이콘 및 단축키(컨트롤+알트+n) 만들기

이 옵션을 선택하면 NVDA를 실행시키는 바로가기 아이콘을 바탕화면에 만들 수 있습니다. 만약 바탕화면 아이콘을 만들었다면 이 아이콘은 단축키 control+alt+n을 할당받습니다. 언제든지 이 단축키를 누르면 NVDA를 시작할 수 있습니다.

3.2.4. 휴대용 버전의 환경 설정을 현재 사용자 계정으로 복사하기

이 옵션을 선택하면 현재 실행 중인 NVDA의 사용자 환경 설정을 NVDA 설치 버전의 설정으로 쓰기 위하여 현재 로그온한 사용자 계정에 복사합니다. 이 옵션은 이 시스템의 다른 사용자의 환경 설정을 복사하지도 않고 윈도우즈 로그온 및 다른 보안 화면용 시스템 환경 설정으로 복사하지도 않습니다. 이 옵션은 휴대용 버전에서 설치할 때에만 제공되며 임시 버전에서 설치시 나타나지 않습니다.

3.3. 휴대용 버전 만들기

만약 다운로드 패키지에서 곧바로 휴대용 버전을 만들고자 한다면 '휴대용 버전 만들기' 버튼을 누르세요. 만약 이미 이 대화상자를 닫았거나 NVDA 설치 버전을 실행 중이라면 NVDA 메뉴 - 도구 - 휴대용 버전 만들기 메뉴 항목을 선택하세요.

대화상자가 나타나면 휴대용 버전이 들어갈 장소를 지정할 수 있습니다. 이 장소는 하드디스크의 디렉토리, USB 드라이브 또는 다른 포터블 미디어일 수 있습니다. 또한 로그온된 사용자의 현재 NVDA 환경 설정을 새롭게 만들어지는 휴대용 버전으로 복사할 수도 있습니다. 이 기능은 설치 버전에서 휴대용 버전을 제작할때에만 제공되며 임시 버전에서 휴대용 버전 제작시 제공되지 않습니다. 'Continue(계속)' 버튼을 누르시면 휴대용 버전이 만들어 집니다. 휴대용 버전 만들기가 완료되면 휴대용 버전 만들기가 성공하였다는 메시지가 나타납니다. "OK"를 누르면 이 대화상자가 사라집니다.

4. NVDA 기본 사용 방법

4.1. NVDA 실행하기

NVDA를 인스톨러로 설치한 경우 control+alt+n을 누르거나 시작 메뉴 - 프로그램 - NVDA 메뉴 - NVDA를 선택하면 간단히 NVDA가 실행됩니다. 그밖에 실행 대화상자(윈도우키+R)에서 nvda를 입력하고 엔터키를 눌러도 됩니다. 만약 NVDA를 사용하고 있다면 NVDA는 재시작합니다. 또한 실행 옵션을 사용하여 NVDA 종료 (-q), 추가 기능 미사용 (--disable-addons) 등을 지정할 수 있습니다.

설치 버전 사용시 사용자 설정은 roaming application data 폴더(예: "C:\Users\<user>\AppData\Roaming")에 저장됩니다. 그러나 이것을 local application data로 경로를 변경할 수 있습니다. 자세한 내용은 시스템 환경 옵션을 참조하세요.

휴대용 버전에서 NVDA를 실행하려면 NVDA를 풀어 놓았던 디렉토리로 이동한 후 nvda.exe에서 엔터키를 누르거나 더블 클릭합니다. 만약 NVDA를 사용하고 있다면 NVDA 종료 후 휴대용 버전이 자동으로 실행됩니다.

NVDA가 시작하면 곧바로 맨처음에 점차 높은 음으로 띠리링 소리가 들립니다(그리고 나서 "NVDA가 로딩 중입니다"라고 알려줍니다). 컴퓨터 속도, NVDA를 USB 메모리에서 실행 또는 더 느린 미디어에서 실행하는지에 따라서 NVDA를 시작하는 데 걸리는 시간이 다릅니다. 만약 NVDA의 시작에 제법 긴 시간이 걸린다면 NVDA는 "NVDA 실행 준비중, 잠시만 기다려 주십시오"라고 말합니다.

이러한 소리가 들리지 않거나 윈도우즈 에러 사운드가 들리거나 점차 낮은 음으로 띠리링하는 소리가 들리는 경우 이는 NVDA에 오류가 있다는 의미이며, 개발자에게 버그를 신고할 필요가 있다고 생각하실 수 있습니다. 이럴 때 어떻게 해야 하는지는 NVDA 웹사이트를 참조해 주세요.

4.1.1. NVDA 시작 대화상자

처음 NVDA를 실행하면 NVDA 기능키와 NVDA 메뉴에 관한 몇가지 기초적인 정보를 제공하는 대화상자가 열립니다. (이 주제에 관한 아래의 섹션을 참고해 주세요.) 또한 이 대화상자는 1개의 콤보박스와 3개의 체크박스를 포함합니다. 콤보박스를 사용하여 사용하고자 하는 키보드 레이아웃을 선택합니다. 첫째 체크박스는 caps lock 키를 NVDA 기능키로 사용할것인지를 묻습니다(체크하면 사용 가능). 두번째 체크박스는 윈도우 로그인 후 NVDA를 실행할것인지를 묻습니다(설치 버전에만 적용됨). 세번째 체크박스는 NVDA가 실행될 때마다 이 대화상자를 보여줄것인지를 묻습니다 (체크시 보여줍니다).

4.1.2. 데이터 사용 통계 대화창

NVDA 2018.3 버전에서부터는 사용자에게 사용 데이터를 NV Access로 보내서 향후 NVDA를 개선하는 데 쓰이도록 하겠는지를 물어봅니다. NVDA를 처음 시작할 때, NVDA를 쓰는 동안 NV Access로 데이터를 보내는 것을 허용할지를 묻는 대화창이 나타납니다. NV Access가 수집하는 데이터에 대한 더 자세한 정보는 일반 설정 영역의 NVDA 프로젝트가 NVDA 사용 통계를 수집하도록 허용 참고: "예"나 "아니오" 버튼을 누르면 이 설정이 저장되며, NVDA를 다시 설치하지 않는 한 이 대화창은 다시 나타나지 않습니다. 하지만, 데이터 수집 과정은 나중에 NVDA의 일반 설정 패널에서 직접 활성화시키거나 비활성화시킬 수 있습니다. 설정을 바꾸려면 NVDA 프로젝트가 NVDA 사용 통계를 수집하도록 허용 체크박스를 조작하면 됩니다.

4.2. NVDA 단축키에 관하여

4.2.1. NVDA 키

NVDA 단축키는 대부분 NVDA 키라 불리는 특정한 키와 하나 이상의 다른 키의 조합으로 구성됩니다. 여기에서 예외는 numpad키만을 단독으로 이용하는 텍스트 리뷰 단축키(데스크탑 레이아웃 사용시) 외 여럿입니다.

NVDA에서는 numpad 인서트키나 확장 인서트키 혹은 caps lock키를 NVDA키로 설정할 수 있습니다. 초기값으로 numpad 인서트키와 확장 인서트키가 NVDA키로 설정되어 있습니다.

만약 NVDA키 중 하나를 원래의 키로 동작시키려면(예를 들어 caps lock키를 NVDA키로 설정한 상태에서 caps lock키를 on으로 켜려고 하는 경우) 그 키를 연속하여 재빨리 두차례 누르세요.

4.2.2. 키보드 레이아웃

NVDA는 데스크탑 레이아웃과 랩탑 레이아웃의 두 가지 단축키 세트를 가지고 있습니다. NVDA는 초기값으로 데스크탑 레이아웃을 사용할 수 있도록 설정되어 있지만 NVDA 메뉴 - 설정 - NVDA 설정 - 키보드 설정에서 랩탑 레이아웃으로 전환 가능합니다.

데스크탑 레이아웃은 (num lock 오프 상태에서) numpad 키들을 빈번하게 사용합니다. 대부분의 랩탑은 물리적인 numpad를 갖고 있지 않지만 일부 랩탑은 Fn키를 누르고 있는 상태에서 키보드의 오른손 부분의 글자와 숫자키를 누름으로써 (예를 들어 (7 8 9 u i o j k l 등) numpad를 모방할 수 있습니다. 만약 여러분이 가진 랩탑이 이렇게 할 수 없거나 num lock를 오프할 수 없다면 랩탑 레이아웃으로 전환해도 좋을 것입니다.

4.3. NVDA 터치 기능에 관하여

만약 터치스크린을 장착한 기기를 사용하시고 그 컴퓨터에 윈도우 8 이상이 설치된 경우 터치 기능을 통해 NVDA를 제어할 수 있습니다 NVDA가 동작하는 동안 터치 지원이 꺼져있지 않는한 모든 터치 입력은 NVDA가 처리합니다. 그러므로 NVDA 비동작시 사용되던 터치 기능은 사용할 수 없습니다. 그럼으로 NVDA가 실행하지 않는 상태에서 동작하는 터치 명령은 실행되지 않습니다. 터치 명령 지원을 토글하려면 NVDA+control+alt+t를 누르십시오. 또한 터치 기능 설정에 있는 터치 명령 지원 설정을 통해 터치 기능을 토글할 수 있습니다.

4.3.1. 화면 탐색

사용자는 터치 기능을 통하여 화면 텍스트 및 여러 컨트롤에 대한 정보를 확인할 수 있습니다 화면 텍스트나 컨트롤 정보를 확인하려면 화면 어디에든지 한손가락으로 터치하십시요. 또한 손가락을 움직이면서 손가락 밑에 있는 정보를 확인할 수 있습니다.

4.3.2. 터치 명령들

본 설명서를 읽으면서 가끔식 NVDA가 사용하는 터치 명령을 접하실 수 있습니다. NVDA에서 사용하는 터치 제스추어 및 그 제스추어의 사용 요령은 다음과 같습니다.

태핑

터치스크린 어디에든지 한개 이상의 손가락으로 태핑하십시요.

한손가락으로 태핑하면 "한손가락 태핑", 두손가락으로 태핑하면 "두손가락 태핑" 등으로 처리합니다.

또한 한개 이상의 손가락으로 여러번 태핑할 수 있는데 이것을 "다중태핑"이라고 합니다. 즉 여러 손가락으로 두번 태핑아면 "두번 태핑", 세번 태핑하면 '세번 태핑" 등으로 처리됩니다. NVDA 사용시 여러 손가락을 이용한 다중 태핑도 가능합니다 (예: 두손가락으로 두번 태핑, 네손가락으로 한번 태핑 등).

쓸어내리기

한개 이상의 손가락을 화면 어디에서든지 쓸어내리는 동작을 의미합니다.

쓸어내리는 방향에 따라 "위로 쓸어내리기", "아래로 쓸어내리기", "왼쪽으로 쓸어내리기"와 "오른쪽으로 쓸어내리기"의 네가지 동작이 있습니다.

태핑과 마찬가지로 여러 손가락을 사용하여 쓸어내릴 수 있습니다 (예: 두손가락으로 위로 쓸어내리기, 네손가락으로 왼쪽으로 쓸어내리기 등).

4.3.3. 터치 모드

수많은 NVDA 명령을 터치 제스추어에 다 할당할 수 없으므로 NVDA에서는 텍스트와 객체 모드의 터치 모드로 화면 탐색 명령을 구분하고 있습니다. 대부분의 터치 명령을 나열할때 어느 모드(텍스트 또는 객체)모드에서 동작하는지를 대괄호 안 내용으로 표시합니다 (예: 위로 쓸어올리기[텍스트 모드]란 이 명령은 텍스트 모드에서만 동작함을 의미함). 만약 터치 명령에 터치 모드가 표시되지않은 경우 아무 모드에서나 동작함을 의미합니다.

터치 모드를 토글하려면 세손가락으로 한번 태핑하십시요.

4.3.4. 터치 키보드

터치스크린 사용시 터치 키보드를 통해 글자 또는 명열을 입력할 수 있습니다. 편집창을 만났을때 터치 키보드를 활성하려면 화면 하단에 위치한 터치 키보드 아이콘을 두번 태핑합니다. 마이크로소프트 Surface Pro와 같은 테블릿을 사용할때 하드웨어 키보드가 연결되어 있지 않다면 터치 키보드가 화면 하단에 항상 표시됩니다. 터치 키보드를 닫으려면 터치 키보드 아이콘을 두번 태핑하거나 포커스를 편집창에서 옮기십시오.

터치 키보드 입력 방식을 사용하려면 터치 키보드를 찾은 후(보통 화면 하단에 위치함) 한 손가락으로 터치 키를 탐색합니다. 원하는 키를 찾았으면 NVDA 설정내 터치 기능 설정에서 설정한 값에 따라 두번 태핑하거나 손가락을 화면에서 뗍니다.

4.4. 입력 도움말 모드

많은 단축키들은 이 사용자 안내서의 나머지 부분에서 설명되겠지만 다양한 단축키 전부를 탐색할 수 있는 간단한 방법은 입력 도움말을 사용하는 것입니다.

입력 도움말을 실행하려면 NVDA+1을 누릅니다. 입력 도움말을 종료하려면 NVDA+1을 한번 더 누르세요. 입력 도움말 실행중 어떤 키를 눌렀을때 (그 키가 정말로 어떠한 기능을 갖고 있다면) 그 키의 기능을 알려 줍니다. 입력 도움말 모드가 켜져 있는 동안 눌러지는 키들은 실제로 그 기능을 수행하지 않습니다. 그래서 어떤 키라도 눌러 보실 수 있습니다.

4.5. NVDA 메뉴

NVDA 메뉴에서는 설정을 제어하고, 도움말을 열고, 환경 설정을 저장하거나 되돌리고, 발음 사전을 수정하고, 부가적인 도구를 접근하고 NVDA를 종료할 수 있습니다.

NVDA가 실행되고 있는 동안 윈도우의 어느 곳에서든 NVDA 메뉴를 호출하고 싶다면 NVDA+N을 누르세요. 또한 윈도우 시스템 트레이를 통해서도 NVDA 메뉴를 불러 낼 수 있습니다. 시스템 트레이에 위치한 NVDA 아이콘을 오른쪽 클릭하거나 윈도우키+B를 눌러 시스템 트레이 목록에서 NVDA 아이콘까지 아래 방향키를 누른후 대부분의 키보드에서 오른쪽 control키 바로 옆에 있는 애플리케이션 키를 누르세요. NVDA 메뉴가 열리면 방향키를 이용하여 메뉴를 탐색할 수 있고 엔터키를 눌러 메뉴 항목을 활성화할 수 있습니다.

5. NVDA로 화면 탐색하기

여러분은 NVDA를 이용하여 표준적인 조작(normal interaction)과 리뷰를 포함하여 몇가지 방식으로 시스템을 탐색하고 이동할 수 있습니다.

5.1. 객체

각 프로그램과 운영체제 그 자체는 많은 객체들로 구성됩니다. 객체란 단일 항목(single item )입니다. 예를 들면 글자 한 개, 버튼 한 개, 체크박스 한 개, 슬라이더 한 개, 목록 한 개 혹은 편집창 한 개 등입니다.

5.2. 시스템 포커스로 탐색하기

포커스라고 불리는 시스템 포커스는 키보드에서 입력한 키를 받아들이는 객체입니다. 예를 들어 편집창에서 글자를 편집하는 경우 그 편집창은 포커스를 갖습니다.

NVDA로 윈도우를 탐색하는 가장 일반적인 방법은 표준 윈도우 키보드 명령을 사용하여 간단하게 시스템 포커스를 이동시키는 것입니다. 예를 들어 탭이나 쉬프트+탭을 눌러서 시스템 포커스를 앞 컨트롤이나 뒤 컨트롤로 이동하기, 알트키를 눌러서 메뉴바를 불러내고 방향키를 이용하여 메뉴를 탐색하기, 알트+탭을 눌러서 실행 중인 프로그램간 이동하기 등입니다. 여러분이 이러한 명령을 사용할때 NVDA는 포커스된 객체의 이름, 유형, 값, 상태, 설명, 단축키, 위치 등 다양한 정보를 알려줍니다. 포커스 커서 표시 기능 사용시 포커스 위치가 화면에 표시됩니다.

시스템 포커스를 움직일 때 유용한 단축키는 다음과 같습니다.

이름 데스크탑 키 랩탑 키 설명

제목 읽기 NVDA+t NVDA+t 현재 사용중인 프로그램의 제목을 알려줍니다. 두 번 누르면 정보를 글자 단위로 풀어 읽고, 세 번 누르면 정보를 클립보드에 복사합니다.
사용 중인 윈도우 읽기 NVDA+b NVDA+b 현재 사용 중인 윈도우의 모든 컨트롤을 읽습니다.(대화상자에서 유용합니다)

5.3. 시스템 케럿으로 탐색하기

텍스트를 탐색하거나 편집할 수 있는 객체가 포커스된 경우, 편집 커서라고 불리기도 하는 시스템 케럿을 사용하여 텍스트 사이를 이동할 수 있습니다.

포커스가 시스템 케럿을 가진 객체에 있는 경우 방향키, 페이지 업키, 페이지 다운키, 홈키, 엔드키 등등을 사용하여 텍스트 사이를 이동할 수 있습니다. 컨트롤이 편집을 지원한다면 텍스트를 편집할 수도 있습니다. NVDA는 글자 단위, 단어 단위, 줄 단위로 이동할때 그리고 텍스트를 선택하거나 선택해제할때 텍스트 내용을 읽어줍니다.

5.4. 객체 탐색

대부분의 프로그램 사용시 포커스 및 케럿을 이동시키는 단축키를 사용합니다. 그러나 가끔은 포커스나 케럿의 이동 없이 사용중인 프로그램이나 운영체제를 탐색하고 싶을 때도 있습니다. 일반적으로 키보드를 이용하여 접근할 수 없는 객체들를 가지고 작업해야 할 때도 있습니다. 이런 경우에 객체 탐색을 사용할 수 있습니다.

객체 탐색은 개별적인 객체들 간을 이동하면서 정보를 얻는 기능입니다. 만약 어떤 객체로 이동했다면 NVDA는 시스템 포커스를 알려주는 것과 마찬가지로 그 객체의 정보를 알려줍니다. 텍스트가 화면에 나타날 때 모든 텍스트를 리뷰하고자 한다면 객체 탐색 대신 화면 탐색 모드를 사용할 수 있습니다.

시스템의 모든 단일 객체들 사이를 전/후로 이동해야 하지만 객체는 상하위 계층적으로 구성되어 있기도 합니다. 즉 어떤 객체(상위 객체)는 다른 객체(하위 객체)를 포함하기 때문에 특정 하위 객체에 접근하려면 그 하위 객체를 지닌 상위 객체 속으로 들어가야 합니다. 예를 들면 목록은 목록 항목을 포함합니다. 그래서 목록 항목에 접근하려면 먼저 그 항목이 나열된 리스트로 이동해야 합니다. 만약 목록 항목으로 이동한 경우 다음이나 이전 객체로 이동한다면 동일한 리스트 안의 다른 목록 항목으로 갈 수 있습니다. 목록 항목의 상위 객체로 이동하면 리스트로 되돌아 갈 수 있습니다. 그 밖에도 만약 다른 객체에 접근하고 싶다면 건너 뛰어 이동할 수 있습니다. 다른 예로 대부분의 프로그램의 툴바는 하위 객체로 각종 컨트롤들을 포함합니다. 그래서 툴바에 위치한 컨트롤로 이동하려면 먼저 툴바로 이동해야 합니다.

일반적으로 현재 리뷰(즉 현재 접근 및 탐색)하고 있는 객체는 탐색 객체(navigator object)라고 불립니다. 객체 탐색 모드 mode를 사용하여 특정 객체를 탐색할때 텍스트 리뷰 단축키를 사용하여 그 객체의 내용을 확인할 수 있습니다. 포커스 커서 표시 기능 사용시 탐색 객체 위치가 화면에 표시됩니다. 초기값으로 탐색 객체는 시스템 포커스를 따라서 이동합니다만 이러한 동작은 온/오프로 토글할 수 있습니다.

기존 고정 리뷰 대신 NVDA의 텍스트 리뷰 명령을 사용하여 현재 탐색 객체 (navigator object), 문서 (document) 또는 화면 (screen)의 내용을 확인할 수 있습니다. 이 탐색 방식을 탐색 모드(review modes)라고 합니다.

다음의 단축키들은 탐색 모드를 전환할 때 사용합니다.

5.6.1. 객체 탐색 모드

객체 탐색 모드에서는 현재 탐색 객체의 내용만 확인할 수 있습니다. 편집창과 같은 기본 컨트롤에서는 그 컨트롤의 내용을 읽을 수 있지만 그 외 객체에서는 그 객체의 이름 또는 값만 확인할 수 있습니다.

5.6.2. 문서 탐색 모드

현재 탐색 객체가 웹페이지나 Lotus Symphony 문서와 같은 복합 문서에 있을 경우 문서 탐색 모드로 문서의 내용을 확인할 수 있습니다.

객체를 탐색하다가 문서 탐색 모드로 전환시 리뷰 커서는 현재 탐색 객체의 위치에 있게 되며 문서를 읽을시 탐색 객체는 리뷰 커서 위치로 옮겨집니다.

참고로 브라우즈 모드를 사용하여 문서를 읽을시 NVDA는 객체 탐색에서 문서 탐색 모드로 자동 전환합니다.

5.6.3. 화면 탐색 모드

현 프로그램 창의 내용을 읽고자 할때 화면 탐색 모드를 사용합니다. 이 기능은 타 스크린 리더들의 화면 읽기 (읽기 포인터) 기능 및 마우스 커서 기능과 유사합니다.

객체를 탐색하다가 화면 탐색 모드로 전환시 리뷰 커서는 현재 화면 위치에 있는 탐색 객체에 위치하게 되며 미뷰 커서로 화면을 읽을시 현 탐색 객체도 리뷰 커서의 위치에 있는 객체로 움직입니다.

참고로 신형 화면 출력 기술이 적용된 프로그램에서는 화면의 모든 내용을 NVDA로 탐색할 수 없습니다.

5.7. 마우스로 탐색 및 조작하기

마우스를 움직이는 경우 NVDA는 초기값으로 마우스의 움직임에 따라 마우스 포인터 아래에 있는 텍스트를 읽습니다. 현재 마우스가 위치한 컨트롤이 지원한다면 NVDA는 텍스트를 문단 단위로 읽을 수 있습니다. 그러나 몇몇 컨트롤들에서는 오직 줄 단위로만 읽을 수 있습니다.

NVDA는 마우스의 움직임에 따라 마우스 포인터 아래에 있는 객체의 타입을 알려주도록 설정할 수도 있습니다(예를 들어 리스트, 버튼, 등등). 이 기능은 때때로 텍스트가 충분치 못할 때 유용할 수 있습니다.

NVDA는 비프음의 변화로 마우스가 스크린의 어느 곳에 위치하는지를 알려줄 수 있습니다. 마우스 포인터를 스크린의 위로 올리면 올릴수록 비프음의 음높이는 올라갑니다. 마우스 포인터를 스크린의 왼쪽이나 오른쪽으로 이동시키면 마우스 포인터의 위치에따라 비프음도 왼쪽 스피커와 오른쪽 스피커 사이에서 들립니다(사용자가 스테레오 스피커나 헤드셋을 가지고 있어야 합니다).

이러한 부가적인 마우스 기능은 초기값으로 설정되어 있지 않습니다. 이 기능을 사용하려면 NVDA 설정에 있는 마우스 설정에서 이 기능들을 설정하세요.

또한 브라우즈 모드를 사용하여 마이크로소프트 워드 문서 일기도 가능합니다.

브라우즈 모드에서 문서의 내용은 평면 방식으로 표현되며 다른 일반 텍스트 문서와 같이 커서키로 텍스트를 이동할 수 있습니다. 따라서 NVDA의 시스템 케럿관련 모든 단축키는 이 모드에서도 사용 가능합니다. 예를 들면 전체 읽기, 서식 알림, 표 탐색 단축키, 등입니다. 포커스 커서 표시 기능 사용시 브라우즈 모드 커서 위치가 화면에 표시됩니다. 특정 텍스트가 링크인지, 헤딩인지 등의 정보는 문서를 이동할 때마다 텍스트와 함께 알려집니다.

가끔씩은 이러한 문서에서 컨트롤들을 직접 조작할 때가 있습니다. 예를 들면 편집창이나 목록에서 글자를 입력하거나 커서키를 사용하여 작업하기 위해서는 컨트롤들을 직접 조작할 필요가 있습니다. 이때 포커스 모드로 전환하여 컨트롤을 조작할 수 있는데 이때 입력된 텍스트나 기능키는 컨트롤 조작용으로 사용됩니다 (예: 글자 입력). 만약 브라우즈 모드일 때 포커스 모드를 필요로 하는 특정 컨트롤을 탭하거나 클릭하는 경우 NVDA는 포커스모드로 자동 전환합니다. 이와 반대로 포커스 모드를 필요로 하지 않는 컨트롤을 탭하거나 클릭하는 경우 브라우즈 모드로 되돌아 옵니다. 포커스 모드를 필요로하는 컨트롤에서 엔터키나 스페이스바를 눌러도 포커스 모드로 전환할 수 있습니다. 이와 반대로 escape 키를 누르면 포커스 모드에서 브라우저 모드로 전환됩니다. 그 밖에도 수동으로 포커스 모드에 강제 진입할 수 있습니다. 포커스 모드로 진입한 후 수동으로 브라우저 모드로 전황할때까지 포커스 모드는 유지됩니다.

이름 키 설명

포커스 모드 사용 종료 escape 이전에 포커스 모드가 자동으로 전환된 경우 브라우즈 모드로 돌아갑니다.
브라우즈 모드 사용시 문서 새로 고침 NVDA+f5 현재 문서의 내용을 다시 불러 옵니다(만약 특정 내용이 문서에서 빠진 것처럼 보이는 경우에 유용합니다; 마이크로소프트 워드 및 아우틀룩은 제외).

6.1. 첫글자로 탐색

브라우즈 모드에서 더욱 빠른 문서 이동을 위하여 NVDA는 문서 안에서 특정 요소로 건너 뛸 수 있는 첫글자 키를 제공합니다. 단 모든 문서가 밑에 나열된 첫 글자 명령을 지원하지 않습니다.

다음의 키는 단독으로 유효한 다음 요소(element)로 이동합니다. 반면에 쉬프트 키와 함께 누르면 이전 요소(element)로 이동합니다.

  • h: 헤딩(heading)
  • l: 목록(list)
  • i: 목록 항목(list item)
  • t: 표(table)
  • k: 링크(link)
  • n: 링크 없는 텍스트(nonLinked text)
  • f: 폼필드(form field)
  • u: 방문하지 않은 링크(unvisited link)
  • v: 방문한 링크(visited link)
  • e: 편집창(edit field)
  • b: 버튼(button)
  • x: 체크박스(checkbox)
  • c: 콤보박스(combo box)
  • r: 라디오 버튼(radio button)
  • q: 인용문/블럭퀏(block quote)
  • s: 구분선(separator)
  • m: 프레임(frame)
  • g: 그래픽(graphic)
  • d: 랜드마크(landmark)

목록이나 표처럼 여러 항목(하위 요소)을 지닌 요소의 처음과 끝으로 이동하려면:

이름 키 설명
컨테이너 요소의 맨 앞으로 이동 shift+comma 케럿이 위치한 컨테이너 요소(목록, 표 등)의 맨 앞으로 이동합니다.
컨테이너 요소의 맨 끝으로 이동 comma 케럿이 위치한 컨테이너 요소(목록, 표 등)의 맨 끝으로 이동합니다.

Gmail, Twitter, Facebook과 같은 웹 앱에서 가끔 첫 글자를 핫키로 사용합니다. 만약 브라우즈 모드에서 커서키를 사용하고자 한다면 NVDA 첫 글자 명령을 임시로 해제시킬 수 있습니다. 특정 문서내에서 첫 글자 명령을 토글하려면 NVDA+shift+space를 누르십시오.

6.2. 요소 목록(Elements List)

요소 목록(elements list)을 이용하여 현재 프로그램에서 불러온 문서별 요소를 확인할 수 있습니다. 예를 들어 웹브라우저에서는 랜드마크, 링크, 폼 필드, 버튼 및 헤딩 목록을 호출할 수 있습니다. 요소 라디오 버튼을 사용하여 확인하고자 하는 요소를 지정할 수 있습니다. 또한 이 대화상자에 있는 편집창을 이용하여 원하는 헤딩, 링크나 랜드마크의 이름을 찾을 수 있습니다 (검색된 내용을 바탕으로 필터링이 됨). 원하는 요소를 찾았다면 이 대화상자에 있는 버튼을 사용하여 선택한 요소 위치로 이동하거나 요소를 활성화할 수 있습니다.

이름 키 설명
브라우즈 모드 요소 목록 NVDA+f7 현재 문서에 적절한 요소 목록을 가져옵니다.

6.4. 임베디드 객체

가끔 여러 페이지들은 Adobe Flash, Oracle Java나 HTML5 또는 에플리케이션과 대화상자같은 리치 컨텐트(rich content)를 포함하는 경우도 있습니다. 이러한 요소들을 마주치면 NVDA는 "임베디드 객체", "에플리케이션" 또는 "대화상자"라고 알려줍니다. 브라우즈 모드 사용시 o 또는 shift+o 첫 글자 명령을 사용하여 본 요소들을 탐색할 수 있습니다. 임베디드 객체를 더 자세히 탐색하려면 엔터키를 누릅니다. 만약 임베디드 객체가 접근 가능하다면 다른 애플리케이션들처럼 탭으로 이동할 수 있고 조작할 수도 있습니다. 임베디드 객체를 포함하는 원래의 페이지로 되돌아가려면 다음의 단축키를 이용하세요.

이름 키 설명
상위 브라우즈 모드 문서로 이동 NVDA+control+space 포커스를 현재 임베디드 객체밖으로꺼내서 그 임베디드 객체를 포함하는 문서로 이동시킵니다.

7. 수학 컨텐츠 읽기

NVDA와 MathPlayer을 이용하여 수학 컨텐츠 읽기 및 접근이 가능합니다. 이를 위해 Design Science의 MathPlayer 4를 사용자 컴퓨터에 설치해야 합니다. MathPlayer를 다운로드하려면 https://www.dessci.com/en/products/mathplayer/을 방문하세요.

NVDA는 다음 수학 컨텐츠를 지원합니다:

  • MathML: 모질라 파이어폭스, 마이크로소프트 인터넷 익스플로러 및 구글 크롬.
  • Design Science MathType: 마이크로소프트 워드 및 파워포인트. MathType가 설치되어 있어야 함. 임시 버전을 사용하셔도 됩니다. https://www.dessci.com/en/products/mathtype/에서 다운로드하여 설치하십시오.
  • Adobe Reader 문서에 삽입된 MathML. 정식 표준이 아님으로 무료로 본 컨텐츠를 읽을 수 있는 프로그램 존재 불확실.
  • Kindle for PC로 불러올 수 있는 책내 수학 컨텐츠 중 접근성이 활성화된 수학 컨텐츠.

수학 컨텐츠가 있는 문서를 읽을때 NVDA는 음성 또는 점자로 수학 컨텐츠를 알려줍니다.

7.1. 집중 탐색

문서를 음성으로 읽을시 가끔 수학 컨텐츠를 만나면 수학 공식 전체보다는 일부분만 읽고싶을때가 있을 수 있습니다.

브라우즈 모드 사용중 수학 컨텐츠를 집중적으로 탐색하고 싶다면 커서를 수학 공식에 맞추고 ENTER를 누르면 됩니다.

만약 브라우즈 모드를 사용하지 않는다면:

  1. 리뷰 커서를 수학 컨텐츠(공식)으로 옮깁니다. 일반적으로 리뷰 커서는 시스템 케럿을 따라가기 때문에 케럿을 수학 컨텐츠로 이동하면 됩니다.
  2. 다음 명령을 실행합니다:

이름 키 설명
수학 컨텐츠 집중 탐색 시작 NVDA+alt+m 수학 컨텐츠 탐색을 시작합니다.

이때부터는 MathPlayer 명령(예: 방향키)을 사용하여 수학 공식을 탐색할 수 있습니다. 예를 들어 좌우 방향키를 사용하여 공식을 읽다가 분수를 자세히 읽고싶을때 아래 방향키로 분수를 확장한 후 방향키를 사용하여 분자와 분모를 읽습니다. 자세한 내용은 MathPlayer 설명서를 참조하세요.

상위 문서로 빠져나가려면 escape키를 누르십시오.

가끔 수학 공식 내용이 대화상자안에 위치하는데 이것을 접근하기 위해 버튼이나 다른 료소가 표시될 수 있습니다. 이때 ctrl+enter를 눌러 공식 정보 버튼이나 요소를 선택합니다.

8.2. 점자 입력

점자 키보드를 사용하여 정자 및 약자 점자 입력이 가능합니다. NVDA 설정내 점자 설정의 점자 입력 표기법 설정을 통해 텍스트 입력시 사용될 점자 코드를 선택할 수 있습니다.

정자 점자로 텍스트 입력 시 점형이 입력되었을 때 곧바로 텍스트가 입력됩니다. 약자 점자를 사용할 때에는 단어의 끝에서 스페이스 또는 엔터를 누르면 텍스트가 입력됩니다. 이때 현재 입력되는 점형은 현재 단어를 입력시에만(즉 앞뒤 단어에는 적용되지 않음) 적용됨을 유의하시기 바랍니다. 예를 들어 숫자를 입력 시 숫자표 점형과 숫자를 입력하고 최신 입력된 숫자를 지우고 새 숫자를 입력할 때 다시 숫자표 점형과 숫자를 입력하여야 합니다.

7점을 눌러 최신 입력된 점형 또는 글자를 지웁니다. 8점을 눌러 엔터를 누르는 동시에 점형을 역점역(텍스트 입력)합니다. 7점과 8점을 눌러 엔터를 누르지 않고 텍스트를 입력합니다.

9. 시각 보조 기능

NVDA는 시각장애인과 저시력 사용자의 음성 및 점자를 통한 컴퓨터 사용을 주 목적으로 하지만 화면 컨텐츠를 사용 용도에 따라 변형시킬 수 있는 기능들을 탑제하고 있습니다. 이 기능들을 NVDA에서는 시각 보조 기능(vision enhancement provider)라고 부릅니다.

NVDA는 밑에서 설명하는 시각 보조 기능들을 제공합니다. NVDA 추가 기능들을 통해 다른 시각 보조 기능들을 설치하고 사용할 수 있습니다.

시각 보조 기능 설정은 NVDA 설정 대화상자에 있는 시각 보조 설정에서 설정할 수 있습니다.

9.1. 포커스 표시

포커스 표시 기능을 통해 시스템 포커스, 탐샛 객체와 브라우즈 모드의 위치를 확인할 수 있습니다.. 이 위치들은 각기 다른 색상의 사각형 모양으로 포시됩니다.

  • Solid blue: 탐색 객체가 시스템 포커스에 위치한 경우(즉 탐색 객체가 시스템 포커스를 따라갈때).
  • Dashed blue: 시스템 포커스.
  • Solid pink: 탐색 객체.
  • Solid yellow: 가상 케럿/브라우즈 모드(웹브라우저내 문서와 같은 일반적인 케럿이 없는 경우).

NVDA 설정 대화상자 dialog에 있는 시각 보조 페널에서 시스템 포커스, 탐색 객체 또는 브라우즈 모드를 표시할지를 설정할 수 있습니다.

9.2. 화면 커튼

시각장애인 사용자로서 화면을 볼 수 없거나 보지 않아도 되는 경우가 입습니다. 또한 다른 사람이 사용자의 화면을 엿본다는 것을 인지하기 쉽지 않습니다. 이를 위해 NVDA는 화면을 까맣게 할 수 있는 "화면 커튼" 기능을 제공합니다.

NVDA 설정 대화상자 dialog에 있는 시각 보조 페널에서 화면 커튼 사용 여부를 설정할 수 있습니다.

10. 문자 인식

어떤 문서나 이미지의 내용을 스크린 리더 사용자가 쉽게 읽을 수 없을때 여러 도구를 사용하여 이미지의 내용을 인식하고 읽을 수 있습니다. 만약 윈도우 10 이상을 사용한다면 NVDA는 윈도우 자체에서 제공하는 문자 인식(OCR)을 활용하여 이미지의 내용을 인식할 수 있습니다. 또한 추가 기능을 통해 더 많은 인식 엔진을 사용할 수 있습니다.

문자 인식 명령을 실행하면 현재 탐색 객체의 내용을 인식합니다. 기본적으로 탐색 객체는 시스템 포커스와 브라우즈 모드 커서를 딸아가기 때문에 대부분의 경우 커서를 원하는 위치에 두고 인식 명령을 실행하면 됩니다. 예를 들어 어떤 사이트를 브라우즈 모드를 사용하여 탐색하다가 그래픽을 만나면 브라우즈 커서가 위치한 그래픽에서 곧바로 인식 명령을 실행하면 됩니다. 그러나 특별한 경우(예: 프로그램 창의 내용을 인식하고 싶을 때) 객체 탐색 명령을 사용하여 원하는 객체로 이동하여 문자 인식 명령을 실행해야 합니다.

문자 인식이 성공하면 인식 내용은 브라우즈 모드 문서와 같은 창에 뜨며 이때 브라우즈 모드와 같은 방법(예: 방향키로 문서 읽기 등)을 사용하여 내용을 확인할 수 있습니다. 인식 결과 문서를 탐색하던 중 스페이스나 엔터를 누르면 커서에 위치한 글자가 포함된 객체가 활성화됩니다(일반적으로 클릭이 됨). 인식 내용을 닫으려면 escape키를 누르세요.

10.1. 윈도우 OCR

윈도우 OCR(여러 언어 지원)과 NVDA를 사용하여 그래픽이나 접근성이 부족한 앱의 내용을 인식할 수 있습니다.

NVDA 설정내 윈도우 OCR 설정을 통해 문자 인식 언어를 선택할 수 있습니다. 인식 언어를 추가하려면 시작 메뉴에서 설정을 선택후 시간 및 언어 > 지역 및 언어 > 언어 추가를 선택합니다.

윈도우 OCR을 사용하여 현재 탐색 객체의 내용을 인식하려면 NVDA+r을 누르십시오.

11.2.2. 요소 목록

마이크로소프트 엑셀에서도 웹과 같은 요소 목록 대화상자를 호출하여 다양한 요소 관련 정보를 확인할 수 잇씁니다. 마이크로소프트 엑셀 사용중 NVDA+f7을 눌러 요소 목록을 호출할 수 있습니다. 요소 목록 대화상자를 통해 다음 요소들을 확인할 수 있습니다:

  • 셀 주석: Worksheet내 주석 목록을 보여줍니다. 주석이 포함된 셀 위치 및 주석 내용 목록이 표시됩니다. 엔터를 누르거나 "옮기기" (move to) 버튼을 클릭하면 선택된 셀로 포커스가 옮겨집니다.
  • 공식: Worksheet내 공식(formula)이 있는 셀을 보여줍니다. 공식이 있는 셀 위치 및 공식 내용 목록이 표시됩니다. 엔터를 누르거나 "옮기기" (move to) 버튼을 클릭하면 선택된 셀로 포커스가 옮겨집니다.
  • Sheet: Workbook내 sheet 목록을 보여줍니다. Sheet의 이름을 변경하려면 sheet를 선택한 후 f2를 누르세요. 엔터를 누르거나 "옮기기" (move to) 버튼을 클릭하면 선택된 sheet로 포커스가 옮겨집니다.
  • 폼 필드: Worksheet내 폼 필드 목록을 보여줍니다. 각 폼 필드의 부제목 및 차지하는 셀 주소가 요소 목록에 나타납니다. 엔터를 누르거나 "옮기기" (move to) 버튼을 클릭하면 폼 필드가 위치한 선택된 셀로 브라우즈 모드 커서가 옮겨집니다.

11.2.3. 셀 노트 알리기

선택된 셀 노트를 확인하려면 NVDA+alt+c를 누르세요. 엑셀 2016과 365 이상 사용시 옛 주석이 "노트"로 변경되었습니다. 요소 목록에서 노트를 선택시 worksheet내 주석을 확인할 수 있습니다.

현재 엑셀에서 제공하는 셀 노트 편집창은 접근성 문제로 제대로 사용할 수 없습니다. 이때문에 NVDA 자체의 셀 노트 편집 대화상자를 제공하며 NVDA를 실행하지 않고도 사용할 수 있도록 엑셀 핫키를 그대로 사용합니다. 포커스된 셀 노트를 편집하려면 shift+f2를 누르세요.

본 핫키는 핫키 설정 대화상자에 나타나지 않으며 다른 핫키로 변경할 수 없습니다.

참고로 셀 팝업 메뉴에서 셀 노트 편집창을 열 수 있습니다. 이때에는 NVDA 자체 대화상자가 아닌 엑셀 편집창이 열립니다.

마이크로소프트 오피스 2016과 365 이상 사용시 새 스타일의 주석 대화상자가 추가되었습니다. 이전 주석 기능과 달리 접근성이 향상되었고 더 많은 기능이 추가되었습니다(예: 주석 답하기 등). 이 대화상자는 포커스된 셀 팝업 메뉴로도 열 수 있습니다. 참고로 이렇게 추가된 주석은 이전 노트와는 별계입니다.

11.2.4. 보호된 셀 내용 확인하기

만약 workbook이 보호된 경우 잠겨진 셀로 이동이 불가능할 수 있습니다. 잠겨진 셀을 탐색하려면 먼저 NVDA+space를 눌러 브라우즈 모드로 전환한 후 엑셀 명령(예: 방향키)을 사용하여 잠겨진 셀을 탐색하십시오.

11.2.5. 폼 필드

엑셀 worksheet내 폼 필드가 존재할 수 있습니다. 이런 경우 요소 목록이나 f/shift+f 폼 필드 첫 글자 명령을 사용하여 폼 필드로 이동할 수 있습니다. 필드 형식에 따라 브라우즈 모드를 사용하여 접근후 엔터를 누르거나 포커스 모드로 전환하여 필드를 접근할 수 있습니다. 브라우즈 모드 및 첫 글자 명령에 대해서는 부라우즈 모드 섹션.을 참조하세요.

12. NVDA 설정하기

대부분의 NVDA 설정은 NVDA 메뉴/설정 메뉴에 위치한 설정별 대화상자를 사용하여 변경할 수 있습니다. 대부분의 옵션은 NVDA 설정 대화상자에서 변경할 수 있습니다(본 대화상자는 다중 페이지 대화상자임). 모든 NVDA 설정 대화상자에서 확인(OK) 버튼을 누르면 변경 내용이 적용됩니다. 변경 내용을 취소하려면 취소(Cancel) 버튼이나 escape 키를 누릅니다. 특정 대화상자에서는 적용(Apply) 버튼을 사용하여 대화상자를 닫지 않고도 변경된 설정을 곧바로 적용할 수 있습니다. 이하의 섹션에서 열거된 일부 설정은 단축키를 사용하여 설정값을 변경할 수 있습니다.

12.1. NVDA 설정

본 대화상자를 통해 대부분의 NVDA 설정을 변경할 수 있습니다. 이 대화상자를 열면 설정 페널 목록이 나타납니다. 설정 페널을 선택할때마다 페널에 관련된 설정이 나타납니다. 설정을 변경한 후 대화상자를 닫지않고 곧바로 적용하려면 적용(Apply) 버튼을 누르세요. 변경된 설정을 적용하고 대화상자를 닫으려면 확인(OK) 버튼을 누르세요.

대부분의 설정 페널은 단춧키(키보드 명령, 터치 명령 등)이 할당되어 있습니다. 설정 페널 호출 명령을 실행하면 NVDA 설정 대화상자가 열리면서 선택된 페널 설정이 보여지게 됩니다. 참고로 모든 설정 페널을 키보드 핫키로 열 수 없습니다. 핫키가 할당되어 있지않는 설정 페널을 호출하려면 먼저 단춧키 설정 대화상자를 통해 핫키를 설정하시기 바랍니다.

NVDA 설정에 나열된 설정 페널들을 알아보겠습니다.

12.1.1. 일반 (NVDA+control+g)

일반 설정을 통해 인터페이스 언어, 업데이트 자동 확인 등 NVDA의 전체적인 기능을 조절할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

NVDA 사용 언어

이 콤보박스에서 NVDA의 인터페이스 및 메시지 표시 언어를 선택할 수 있습니다. 기본값을 포함하여 NVDA가 지원하는 언어들을 나열하고 있습니다. 초기 설정값은 "기본값, Windows"이며 이 옵션을 선택하면 NVDA는 윈도우에서 설정된 디스플레이 언어를 사용하여 메시지를 알려줍니다.

주의: 사용 언어를 변경하려면 반드시 NVDA를 재시작하여야 합니다. 확인 대화상자에서 새로 설정한 언어를 지금 또는 나중에 사용하기 위해 "지금 재시작" 또는 "나중에 재시작"중 선택할 수 있습니다. 만약 "나중에 재시작"을 선택한 경우 먼저 환경 설정을 저장(수동 또는 NVDA 종료시 자동 저장)하시기 바랍니다.

종료 시 NVDA 환경설정 저장

이 체크상자를 체크하면 NVDA 종료시 현재 환경 설정이 자동으로 저장됩니다.

종료 시 NVDA 종료 옵션 표시

이 옵션은 NVDA 종료시 종료 옵션 대화상자를 띄울것인지를 조절합니다. 이 옵션을 선택하면 NVDA 종료전 종료 옵션(종료, 재시작, 재시작 및 추가 기능 미사용, 업데이트 설치)을 묻는 대화상자가 나타납니다. 본 옵션을 해제하면 NVDA 종료 명령을 누르면 NVDA는 곧바로 종료됩니다.

NVDA 시작 및 종료음 재생

이 체크상자를 체크하면 NVDA 시작 및 종료시 시작 및 종료음이 재생됩니다.

로깅 레벨

이 콤보박스에서 NVDA 실행중 로그 기록 불량을 설정할 수 있습니다. 일반적으로 사용자들은 지나치게 많은 양을 기록하도록 이 옵션을 건드릴 필요가 없을 것입니다. 그러나 개발자에게 버그 리포트 정보를 제공하거나 로깅을 켜거나 끌 때 이 옵션이 유용합니다.

현재 사용할 수 있는 로깅 레벨은 다음과 같습니다:

  • 로깅 않함: 간단한 시작 메시지 외에 로깅 기록이 남지 않습니다.
  • 정보: 시작 메시지나 개발자에게 유용한 메시지가 기록됩니다.
  • 디버그 주의: 심한 오류가 아닌 주의 메시지가 기록됩니다.
  • 입출력: 키보드 및 점자디스플레이 입력과 음성 및 점자 출력이 기록됩니다.
    • 만약 개인 정보 보호에 민감하다면 이 옵션을 피하십시오.
  • 디버그: 위에 나열한 기록(정보, 주의, 입출력) 이상의 디버그 내용이 기록됩니다.
    • 입출력 기록과 마찬가지로 개인 정보 보호에 민감하다면 이 옵션을 피하십시오.

로그인 후에 NVDA 시작

이 옵션을 선택하면 NVDA는 윈도우즈에 로그온 후 자동으로 실행됩니다. 이 옵션은 NVDA 설치 버전에서만 유효합니다.

로그인 중에 NVDA 사용 (관리자 권한 필요)

이 옵션을 선택하면 NVDA는 로그온 화면에서 (예: 사용자 이름과 암호를 입력하여 윈도우즈에 로그온할 때) 자동으로 실행됩니다. 이 옵션은 NVDA 설치 버전에서만 유효합니다.

Windows 로그인 및 보안 화면에서 현재 저장된 설정 사용 (관리자 권한 필요)

이 버튼을 누르면 현재 저장된 NVDA 사용자 환경 설정을 NVDA 시스템 환경 설정 디렉토리에 복사합니다. 이후 갓 복사된 사용자 환경 설정은 윈도우즈 로그온 화면이나 사용자 계정 컨트롤(UAC) 및 기타 보안 화면에서 NVDA 사용시 적용됩니다. 모든 설정값들을 확실하게 복사하려면 먼저 control+NVDA+c를 누르거나 NVDA 메뉴에서 환경 설정을 저장하세요. 이 옵션은 NVDA 설치 버전에서만 유효합니다.

NVDA에서 업데이트를 자동으로 확인

이 옵션을 선택하면 NVDA는 자동으로 업데이트 버전을 체크하며 업데이트가 있을때 업데이트 대화상자를 띄웁니다. 또한 NVDA 메뉴 - 도움말 - 업데이트 확인을 선택하여 수동으로 업데이트를 체크할 수 있습니다. 자동 또는 수동으로 업데이트 확인시 사용자의 시스템에 적절한 업데이트를 제공하기 위해 NVDA 관련 정보가 업데이트 서버에 전송됩니다. 다음 정보는 항상 전송됩니다:

  • 사용중인 NVDA 버전
  • OS 버전
  • 32비트 또는 64비트 OS

==== NVDA 프로잭트에 NVDA 사용 통계를 전송하도록 허용 ====[GeneralSettingsGatherUsageStats][GeneralSettingsGatherUsageStats] 본 옵션을 선택시 NVDA 업데이트 확인시 사용자 현황(사용 국가 등)이 NV Access에 전송됩니다. 참고로 IP 주소를 통해 사용 국가는 확인할 수 있어도 IP 주소 자체는 업데이트 서버에 저장되지 않습니다. 업데이트 확인에 필요한 정보 외에도 다음 데이터가 전송됩니다:

  • NVDA 인터페이스 언어
  • NVDA 설치 여부(설치용 또는 휴대용)
  • 사용중인 음성 엔진(추가 기능 포함) 이름
  • 사용중인 점자디스플레이(추가 기능 포함) 이름
  • 점자 출력표(점자 사용중일때)

NV Access는 본 정보를 추후 개발 반영 목적으로 사용합니다.

NVDA 실행시 설치 준비중인 업데이트 알림

본 옵션을 선택하면 NVDA 업데이트가 준비되어있고 NVDA를 재시작할 경우 업데이트 설치 관련 대화상자가 나타나게 됩니다. 또한 NVDA 종료 대화상자(만약 그 옵션을 선택한 경우)내 업데이트 설치 또는 도움 메뉴에 있는 업데이트 확인을 통해서 새 버전을 설치할 수 있습니다.

12.1.2. 보이스 (NVDA+control+v)

NVDA 설정내 보이스 설정 페널은 음성 엔진 변경 및 음성의 속성을 변경할 수 있는 옵션들을 제공합니다. 이와 별도로 어느 곳에서도 음성 매개변수를 제어할 수 있는데 이에 대해서는 간단 음성엔진 설정 섹션을 참조하세요.

보이스 설정 페널에서 설정할 수 있는 옵션은 다음과 같습니다.

음성 엔진 변경

보이스 설정의 첫 컨트롤인 "변경" 버튼을 누르면 음성 엔진 선택이 NVDA 설정 위에 열립니다. 음성 엔진 선택 대화상자에서는 사용하고자 하는 엔진과 출력 장치를 설정할 수 있습니다. 음성 엔진 설정을 적용하거나 변경을 취소하면 NvDA 설정 대화상자로 돌아오게 됩니다.

본 대화상자의 첫 옵션은 시스템에 설치되어 있는 현재 음성엔진의 모든 보이스들을 열거한 콤보박스입니다. 방향키를 이용하여 다양한 보이스를 선택하여 들어 볼 수 있습니다. 왼쪽방향키나 위쪽방향키를 누르면 리스트에서 위로 이동하고, 반면 오른쪽방향키나 아래쪽방향키를 누르면 리스트에서 아래로 이동합니다.

보이스 버전 (Variant)

만약 NVDA에 내장된 Espeak NG 음성엔진을 사용 중이라면 이 콤보박스에서 음성엔진의 보이스 버전(음색)을 선택할 수 있습니다. ESpeak NG의 음색은 비슷비슷합니다. 왜냐하면 그 보이스들이 약간 상이한 속성들을 eSpeak 보이스에 제공하기 때문입니다. 어떤 음색은 남성처럼 들리고, 어떤 음색은 여성처럼 들리며 심지어 어떤 음성은 개구리처럼 들립니다. 만약 타 음성 엔진이 지원시 이 설정을 변경할 수 있습니다.

음성 속도

이 옵션에서는 사용자가 선택한 음성의 속도를 변경할 수 있습니다. 이것은 0에서 100까지 움직이는 슬라이더입니다. (0은 가장 느리고 100은 가장 빠릅니다.)

음성 높이

이 옵션에서는 현재 보이스의 높이를 변경할 수 있습니다. 이것은 0에서 100까지 이동하는 슬라이더입니다. (0은 가장 낲은 높이이고, 100은 가장 높습니다.)

음성 크기

이 옵션은 0에서 100까지 이동하는 슬라이더입니다. (0은 가장 작은 소리이고, 100은 가장 큽니다.)

억양

이 옵션은 (음높이를 얼마나 높이고 낮출지) 음성엔진이 사용할 억양의 양을 선택하는 슬라이더입니다.

음성 엔진 지원시 자동으로 언어 전환

이 체크박스를 체크하면 읽고 있는 텍스트에서 언어 markup이 유효한 경우 NVDA는 자동으로 음성엔진의 언어를 전환합니다. 기본으로 이 옵션은 선택되어 있습니다.

음성 엔진 지원시 자동으로 종속어 전환

자동 음성 전환이 유효한 경우 이 체크박스를 통해 단순 언어 변경 뿐 아니라 종속어(사투리) 변경 유무를 토글할 수 있습니다. 예를 들면, 미국식 영어로 문서를 읽고 있는데 일부 텍스트가 영국식인 경우 만약 이 옵션을 선택하면 음성엔진은 액센트를 전환할 것입니다. 기본으로 이 옵션은 해제되어 있습니다.

구두점/기호 출력 레벨

단축키: NVDA+p

이 옵션에서는 단어로 읽어야 할 구두점과 다른 기호들의 출력 레벨을 선택할 수 있습니다. 예를 들면 모두(all)로 설정한 경우 모든 기들을 단어로 읽을 것입니다. 이 옵션은 현재 사용중인 음성엔진 뿐만 아니라 모든 음성엔진에 적용됩니다.

==== 기호 및 글자 출력시 선택된 음성 엔진 언어 사용 ====[SpeechSettingsTrust][SpeechSettingsTrust] 이 옵션을 선택(기본으로 선택되어 있음)시 기호와 글자를 읽을때 현재 선택된 음성 엔진의 언어 데이타를 사용하여 출력하게 됩니다. 만약 기호 및 글자 출력 언어가 선택된 엔진 언어와 다를 경우 이 옵션을 해제하여 NVDA 전체 언어 설정을 사용하도록 하시면 됩니다.

기호 및 글자 출력 시 Unicode Consortium 데이터(이모지 포함) 사용

이 체크박스를 선택하면 기호 및 글자 출력시 추가 발음 사전을 인용하게 됩니다. 특정 기호(예: 이모지)를 출력시 Unicode Consortium에서 제공하는 Common Locale Data Repository 사전을 인용하게 됩니다. 만약 이 사전에 들어있는 이모지 설명을 어느 음성 엔진에서나 듣고싶다면 본 옵션을 선택하세요. 그러나 사용하는 음성 엔진이 이모지 데이터를 탑제한 경우 이 체크박스를 해제하세요.

참고로 사용자가 특정 이모지에 대한 발음 설명을 설정한 경우 본 설정에 상관없이 사용자가 지정한 데이터가 출력됩니다(사용자 기호 발음 사전은 사용자 설정 환경에 저장됨). 기호 발음 설정에 대해서는 기호/구두점 설정 대화상자 섹션을 참고하세요.

어느 곳에서나 Unicode Consortium 데이터 설정을 토글하려면 단춧키 설정 대화상자를 통해 핫키를 설정하세요.

대문자 음높이 변경율

이 편집창에서는 대문자를 읽을 때 변경될 보이스의 음높이를 수치로 입력할 수 있습니다. 이 수치는 백분율입니다. 여기에서 음수는 음높이를 낮추고 양수는 음높이를 올립니다. 음높이를 변경하지 않으려면 0을 사용하세요.

대문자 앞에 "cap" 붙여 읽기

이 체크박스가 체크된 경우 NVDA는 글자 단위로 읽을 때 대문자 앞에 "cap" 단어를 붙여서 읽게 됩니다. 일반적으로 NVDA는 음높이를 약간 올려서 대문자를 읽습니다. 그러나 일부 음성엔진들은 이 기능을 제대로 지원하지 않으며 이럴 경우 본 옵션이 유용합니다.

대문자 앞에 비프음 출력

이 체크박스가 체크된 경우 NVDA는 글자 단위로 대문자를 만날 때마다 작은 비프음을 냅니다. "대문자 앞에 cap 붙여 읽기" 체크박스처럼 이 옵션은 대문자의 음높이를 변경할 수 없는 음성엔진을 사용할 때 유용합니다.

철자 발음 지원시 사용

일부 단어들은 단지 한 글자인 경우가 있습니다. 그러나 그 글자가 (풀어 읽기처럼) 개별 글자로 발음되는지 그렇지 않으면 단어로 발음되는 지에 따라서 발음이 서로 다릅니다. 예를 들면 영어에서 "a"는 글자이기도 하고 단어이기도 합니다. 이 옵션은 음성엔진이 위의 두가지 경우를 구분할 수 있게 합니다. 단 음성엔진이 이 기능을 지원해야 합니다. 대부분의 음성엔진들은 이 기능을 지원합니다.

이 옵션은 일반적으로 사용 가능합니다. 그러나 일부 마이크로소프트 음성 API 엔진들은 이 기능을 정확하게 지원하지 않아서 이 기능이 유효할 때에도 이상하게 읽습니다. 만약 글자 단위 일기에서 문제가 있다면 이 옵션을 해제해 보세요.

12.1.3. 음성 엔진 선택 (NVDA+control+s)

NVDA 설정내 보이스 페널에 있는 "변경" 버튼을 선택하면 음성 엔진 대화상자가 나타납니다. 음성엔진 대화상자에서는 NVDA가 사용할 음성엔진을 선택할 수 있습니다. 사용하고자 하는 음성 엔진을 선택한 후 확인(OK)를 누르면 NVDA는 선택된 음성 엔진으로 전환합니다. 만약 음성 엔진 적용시 오류가 발생하면 메시지를 띄운 후 이전 엔진을 사용하게 됩니다.

음성 엔진

이 옵션에서는 NVDA에서 사용하고자 하는 음성 엔진을 선택할 수 있습니다.

NVDA가 지원하는 음성엔진의 대해서는 지원하는 음성엔진 섹션을 참조하세요.

특히 이 목록에서 항상 나타나 있는 "음성 출력 끔(No speech)"을 선택하면 아무런 음성 출력없이 NVDA를 사용할 수 있습니다. 이 항목은 단지 점자로만 NVDA를 사용하고자 하는 사용자나 단지 음성 뷰어(Speech Viewer)만을 사용하고자 하는 정안인 개발자에게 유용합니다.

음성 출력 장치

이 옵션에서는 선택된 음성엔진이 음성을 출력할 사운드 카드를 선택할 수 있습니다.

오디오 도킹 모드

Key: NVDA+shift+d

윈도우 8 이상 사용시 배경음 볼륨 조절(오디오 도킹) 시기(NVDA 실행 또는 음성 출력)를 본 옵션을 통해 설정할 수 있습니다.

  • 도킹 않함: NVDA 사용중 배경음 볼륨이 도킹되지 않습니다.
  • 음성 및 소리 출력시 도킹: NVDA가 말할때 또는 사운드를 재생시 도킹(음성 엔진이 지원시)이 적용됩니다.
  • 도킹 사용: NVDA 사용시 배경음 볼륨은 항상 도킹됩니다.

이 옵션은 NVDA 설치 버전에서만 유효합니다. 휴대용 버전 및 임시 버전 사용시 오디오 도킹을 사용할 수 없습니다.

12.1.4. 간단 음성엔진 설정(Synth settings ring)

NVDA는 보이스 설정에 접근하지 않아도 어느 곳에서나 보이스 설정을 변경할 수 있는 단축키들을 제공합니다.

12.1.5. 점자

점자 설정 페널을 통해 점자 입출력 관련 옵션을 설정할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

점자디스플레이 변경

점자 설정의 첫 컨트롤인 "변경" 버튼을 누르면 점자디스플레이 선택 대화상자가 NVDA 설정 위에 열립니다. 점자디스플레이 선택 대화상자에서 원하는 디스플레이를 선택할 수 있습니다. 사용하고자 하는 디스플레이를 선택하거나 변경을 취소하면 NvDA 설정 대화상자로 돌아오게 됩니다.

점자 출력 표기법 (Output Table)

점자 페널의 다음 옵션은 점자 출력 표기법입니다. 이 옵션에서 텍스트를 점자로 변환할때 사용될 표기법을 선택할 수 있습니다. 이 콤보박스에는 다양한 언어, 표준 및 각 그레이드의 점자테이블이 있습니다. 목록 안에 있는 점자 테이블에서 점자테이블로 이동하려면 화살표키를 사용하세요.

점자 입력 표기법 (Input Table)

점자 출력 표기법과는 별도로 이 옵션에서 점자 입력 표기법얼 설정할 수 있습니다. 선택된 표기법은 점자디스플레이에 장착된 점자 키보드에서 텍스트를 입력시 텍스트 변환용으로 쓰입니다. 목록 안에 있는 점자 테이블에서 점자테이블로 이동하려면 화살표키를 사용하세요.

참고로 모든 디스플레이가 점자 입력을 지원하지 않습니다 (점자디스플레이 드라이버에 따라 유무함). 자세한 정보는 지원하는 점자 디스플레이 섹션을 참조하세요.

커서 위치 단어 정자로 보기

이 옵션은 커서 아래의 단어를 약자가 아닌 컴퓨터 점자로 표시할지를 설정합니다.

점자 커서 출력

이 옵션을 사용하여 점자 커서가 출력될지를 설정합니다. 본 설정은 시스템 caret과 리뷰 커서에 적용되며 텍스트 선택에는 적용되지 않습니다.

점자 커서 깜박임

이 옵션을 사용하여 점자 커서 깜박이를 설정합니다. 깜박이가 꺼지면 점자 커서 점형이 디스플레이에 항상 보여지게 됩니다. 텍스트 선택시에는 본 설정이 적용되지 않습니다(7점과 8점이 항상 출력됨).

커서 깜박임 시간

이 수치 필드에서는 커서의 깜박이 속도를 1000분의 1초 단위로 변경할 수 있습니다.

포커스 커서 모양

본 옵션을 사용하여 포커스와 연계되었을때의 점자 커서 모양(점형)을 설정할 수 있습니다. 본 옵션은 텍스트 선택 커서 출력에는 적용되지 않습니다.

리뷰 커서 모양

본 옵션을 사용하여 리뷰 커서와 연계되었을때의 점자 커서 모양(점형)을 설정할 수 있습니다. 본 옵션은 텍스트 선택 커서 출력에는 적용되지 않습니다.

알림 메시지 출력 시간

이 수치 필드는 점자 디스플레이에서 NVDA 메시지가 얼마나 오래 표시될지를 제어합니다. 만약 0초로 설정하면 NVDA 메시지가 점자로 출력되지 않습니다. 점자디스플레이 라우터 버튼을 누르면 NVDA 메시지는 사라지지만 알림 메시지를 보여주는 핫키를 누르면 다시 보여집니다.

점자 메시지를 영구적으로 표시

본 옵션을 선택하면 NVDA 메시지는 점자디스플레이에 영구적(명령 입력 전까지)으로 표시됩니다.

점자 연계 커서 (Tether braille)

단축키: NVDA+control+t

이 옵션에서는 점자 디스플레이가 시스템 포커스, 탐색 객체/리뷰 커서, 또는 둘 다 따라갈 것인지를 선택할 수 있습니다. "자동 연계"를 선택하면 시스템 포커스나 케럿이 변경시 점자 커서가 변경됩니다. 만약 객체 탐색 또는 리뷰 커서 명령을 사용하면 점자는 새 객체나 리뷰 커서 위치에 위치하며 시스템 포커스나 케럿이 변경시 점자는 다시 포커스나 케럿을 따라가게 됩니다. 점자가 포커스와 케럿만 따라가도록 하려면 포커스 커서 연계로 설정해야 합니다. 이때 객체 탐색 또는 리뷰 커서 명령을 실행하면 점자는 따라가지 않습니다. 객체 탐색 또는 리뷰 커서와 점자를 연동하려면 리뷰 연계로 설정해야 합니다. 이때 점자는 포커스와 케럿을 따라가지 않습니다.

문단 단위로 읽기

이 옵션이 선택되면 점자는 줄 단위 대신 문단 단위로 출력됩니다. 따라서 다음 혹은 이전 줄 명령도 문단 단위로 이동합니다. 즉 줄 끝에 점자로 읽을 수 있는 내용이 더 있다면 스크롤 명령을 여러번 실행하지 않아도 됩니다. 이 옵션은 많은 양의 텍스트를 빨리 읽을 때 편리합니다. 기본으로 이 옵션은 해제되어 있습니다.

가능하면 끊지 않고 단어 출력

이 옵션을 선택하면 어떤 단어의 길이가 점자디스플레이 길이를 초과할 경우 그 단어를 끊지않고 읽을 수 있습니다. 이를 워드랩이라고 하며 이때 디스플레이의 일부는 공백으로 처리되고 그 자리에 있던 단어는 디스플레이를 스크롤하면 보여집니다. 그러나 단어의 전체(문자열) 길이가 디스플레이의 길이보다 길 경우 단어는 끊어질 수도 있으니 주의하시기 바랍니다.

만약 본 옵션을 해제하면 긴 단어의 일부만 출력됩니다. 이때 점자를 스크롤하여 나머지 부분을 읽을 수 있습니다.

이 옵션은 대용량 텍스트를 읽을시 유용하나 그만큼 점자를 스크롤해야 한다는 단점이 있습니다.

포커스 세부 내용 출력

이 옵션을 사용하여 포커스된 객체의 세부 내용이 점자로 어떻게 전달될 것인지를 설정합니다. 여기에서 세부 내용이란 포커스를 포함하는 객체의 상하위 객체 관계(hierarchy) 정보를 말합니다. 예를 들어 어떤 목록 항목에 포커스가 옮겨졌을때 그 항목은 목록 객체의 하위 객체로 인식됩니다. 그리고 그 목록은 어떤 대화상자의 컴트롤 중 하나일 수 있습니다. 이를 포함한 NVDA에서의 상하위 객체 탐색에 대한 정보는 객체 탐색 섹션에서 설명하고 있습니다.

만약 "포커스 이동시 출력"을 선택하면 포커스 객체에 대한 세부 내용은 포커스가 다른 객체로 옮겨질때에만 출력됩니다. 위의 목록 항목과 같은 경우 목록으로 포커스를 옮길때 항목 이름이 점자로 출력됩니다. 또한 점자 디스플레이에 왼쪽 부분에 공간이 남는다면 포커스된 객체가 목록 항목이라는 것도 출력됩니다. 그 후 방향키를 사용하여 다른 항목으로 이동할때 NVDA는 포커스가 같은 목록안에 있다고 인지하고 포커스된 항목에 대한 내용만 점자로 출력합니다. 만약 목록 및 그 외 객체(예: 대화상자 내용 등)을 읽고싶으면 점자 디스플레이를 뒤로 스크롤하여야 합니다.

만약 "항상 세부 내용 출력" 옵션(NVDA 2017.2 이하에서 적용됨)을 선택하면 어느 객체로 포커스가 이동하던지 세부 내용이 점자로 출력됩니다. 항상 포커스 객체에 대한 세부 내용을 읽고싶다면 본 옵션을 선택하십시오. 그러나 어느 점자셀에서 포커스 내용이 시작될지가 다르다는 단점이 있습니다. 특히 특정 상황(에: 긴 목록을 탐색할때)에서 손가락을 디스플레이 위에서 항상 움직여야 하는 불편함이 있습니다.

만약 "뒤로 스크롤시에만 출력"을 선택하면 포커스가 다른 객체로 이동해도 세부 내용을 점자로 출력하지 않습니다. 위의 상황에서 본 옵션이 적용되었을때 포커스가 목록으로 이동할때 목록 항목 이름만 점자로 보여줍니다. 그럼으로 목록과 대화상자 관련 세부 내용을 보려면 점자 디스플레이를 뒤로 스크롤해야 합니다.

어느 곳에서나 포커스 객체 세부 내용 출력 옵션을 토글하려면 단춧키 설정 대화상자를 통해 핫키를 설정하십시오.

12.1.6. 점자디스플레이 선택 (NVDA+control+a)

NVDA 설정내 점자 페널에 있는 "변경" 버튼을 선택하면 점자디스플레이 선택 대화상자가 나타납니다. 본 대화상자에서는 사용하고자 하는 디스플레이를 선택할 수 있습니다. 디스플레이를 선택한 후 확인(OK)를 누르면 NVDA는 선택된 디스플레이와의 연결을 시도합니다. 만약 디스플레이와 연결을 시도할때 오류가 발생하면 메시지를 띄운 후 이전 디스플레이를 사용하게 됩니다.

점자 디스플레이

시스템에 어떤 점자 디스플레이가 사용가능한지에 따라서 몇가지 옵션들이 표시됩니다. 방향키를 이용하여 각 옵션들로 이동합니다.

만약 자동 연결(automatic) 옵션을 선택하면 NVDA는 사용할 수 있는 점자디스플레이를 계속 찾게 됩니다. 본 기능을 사용하여 자동 연결이 지원되는 디스플레이를 USB나 블루투스로 연결하면 NVDA는 연결된 디스플레이를 사용하게 됩니다.

점자 출력 끔(No braille)은 사용 가능한 점자 디스플레이가 없다는 의미입니다.

NVDA가 지원하는 점자 디스플레이와 각 디스플레이의 자동 연결 지원에 대한 더 상세한 정보를 원하시면 지원하는 점자 디스플레이 섹션을 참조하세요.

포트 (Port)

사용자가 선택한 점자디스플레이가 여러 연결 포트를 지원한다면 이 콤보박스에 나열된 포트 중 하나를 선택하여 특정 디스플레이와 연결하여 사용할 수 있습니다. 만약 선택한 디스플레이가 "자동 연결" 방식 (아래를 참조 바람)만 지원할 경우 이 옵션은 나타나지 않습니다.

기본적으로 점자디스플레이와 연결시 "자동 연결" 방식이 선택됩니다. 이때 NVDA는 시스템에 설치된 USB 또는 블루투스 장치 중에서 사용자가 선택한 점자 디스플레이를 검색한 후 디스플레이가 인식이 되면 자동으로 연결을 시도합니다.

만약 사용자가 선택한 디스플레이가 자동 연결 방식을 지원하지 않는 경우 특정 포트를 지정할 수 있습니다. 이때 사용자는 자동 연결 (ㅜ이에서 언급한 방식 사용), USB, 블루투스 또는 옛 시리얼 포트 중 하나를 선택하여 특정 디스플레이와 연결을 시도할 수 있습니다.

특정 디스플레이가 지원하는 포트에 대해서는 지원되는 점자디스플레이 섹션을 참조하시기 바랍니다.

12.1.7. 시각 보조

시각 보조 페널을 통해 시각 보조 기능 사용 여부 또는 각 도구별 설정을 변경할 수 있습니다.

참고로 NVDA 추가 기능에서 제공하는 설정을 통해 페널 구성이 달라질 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

포커스 표시

포커스 표시 그루핑에 나열된 체크박스를 통해 NVDA의 포커스 표시 기능을 설정할 수 있습니다.

  • 포커스 표시 사용: 포커스 표시 사용 여부를 설정합니다.
  • 시스템 포커스 표시: 시스템 포커스를 시각적으로 표시할지를 설정합니다..
  • 탐색 객체 표시: 탐색 객체를 시각적으로 표시할지를 설정합니다.
  • 브라우즈 모드 커서 표시: 브라우즈 모드 커서를 시각적으로 표시할지를 설정합니다.

참고로 "포커스 표시 사용" 설정이 다른 체크박스들의 체크 상태를 좌우합니다. 그럼으로 "포커스 표시 사용" 체크박스를 선택하면 다른 3개의 체크박스들도 체크되게 됩니다. 또한 "시스템 포커스 표시"만 선택된(탐색 객체 및 브라우즈 모드 커서는 해제됨) 경우 "포커스 표시 사용" 체크박스는 반만 체크됩니다.

화면 커튼

"화면 커튼 사용" 체크박스를 선택하여 화면 커튼 기능을 실행할 수 있습니다. 이때 화면 커튼 사용시 화면이 까맣게 된다라는 경고 메시지가 나타납니다. 본 기능 사용("ㅖ"버튼을 누르기)전 음성 또는 점자가 사용중이고 화면이 없어도 컴퓨터를 사용할 수 있는지 확인하시기 바랍니다. 화면 커튼을 사용하지 않고싶다면 "아니요"를 선택하십시오. 화면 커튼을 사용하고 싶다면 "예"를 선택하십시오. 이 대화상자에 표시된 설정을 통해 경고 메시지 출력 여부를 선택할 수 있습니다. 항상 경고 메시지를 보려면 "화면 커튼 사용" 체크박스 밑에 있는 "경고 메시지 보이기" 체크박스를 선택합니다.

화면 커튼을 어느곳에서나 토글하려면 단춧키 설정 대화상자를 통해 토글 명령을 설정하세요.

기본적으로 화면 커튼이 토글될때 사운드가 재생됩니다. 사운드 재생 여부는 "화면 커튼 토글시 사운드 재생" 체크박스를 통해 설정합니다.

다른 시각 보조 기능 설정

NVDA 추가 기능을 통해 더 많은 시각 보조 기능을 사용할 수 있습니다. 추가 기능 설정(설정할 수 있는 기능인 경우)은 각 보조 기능별 그루핑에서 확인할 수 있습니다. 자세한 내용은 사용하고자 하는 시각 보조 기능 설명서를 참고하세요.

12.1.8. 키보드 (NVDA+control+k)

키보드 페널을 통해 키보드 사용 및 입력에 대한 NVDA의 반응 관련 옵션을 설정할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

키보드 레이아웃

이 콤보박스에서는 NVDA가 어떤 키보드 레이아웃을 사용할 것인지를 선택할 수 있습니다. NVDA는 데스크탑과 랩탑의 두가지 레이아웃을 제공합니다.

NVDA 기능키

본 체크박스 목록을 통해 NVDA 기능키를 지정할 수 있습니다. 아래 다음 키 목록에서 선택할 수 있습니다:

  • caps lock 키
  • numpad insert 키
  • 확장 insert 키(보통 방향키 위와 home 및 end 키 옆에 위치함)

NVDA 기능키가 지정되지 않은 경우 대부분의 NVDA 명령을 사용할 수 없습니다. 이럴 경우 설정을 닫을시 기능키에 대한 오류 메시지가 뜹니다. 오류 메시지를 닫은 후 키보드 설정에서 최소한 한개 이상의 기능키를 선택하십시오.

입력한 글자 알림

단축키: NVDA+2

이 옵션이 선택되면 NVDA는 키보드에서 입력한 모든 글자를 글자 단위로 읽어줍니다.

입력한 단어 알림

단축키: NVDA+3

이 옵션이 선택되면 NVDA는 키보드에서 입력한 모든 글자를 단어 단위로 읽어줍니다.

텍스트 입력시 음성 정지

이 옵션을 선택하면 텍스트를 입력할때마다 음성이 정지됩니다. 기본으로 이 옵션은 선택되어 있습니다.

엔터키 입력시 음성 정지

이 옵션을 선택하면 엔터키를 누를때마다 음성이 정지됩니다. 기본으로 이 옵션은 선택되어 있습니다.

모두 읽기 사용시 건너뛰어 읽기 사용

이 옵션이 선택되면 모두 읽기 도중 다음이나 이전 줄, 문단 또는 웹 요소로 이동하여도 모두 읽기가 정지되지 않습니다.

caps lock이 켜져 있을때 소문자 입력시 경고음 출력

이 옵션이 선택된 후 만약 caps lock이 켜져 있고 쉬프트와 동시에 글자키가 입력되었다면 경고 비프음이 출력됩니다. 일반적으로 caps lock이 켜져 있고 쉬프트와 동시에 글자키가 입력된 경우는 의도적으로 발생하지 않으며 caps lock이 켜져있다는 사실을 자각하지 못한데서 기인합니다. 그러므로 이것을 경고하는 것은 매ㅜ 유익할 수 있습니다.

기능키 알림

단축키: NVDA+4

이 옵션이 선택되면 NVDA는 키보드에서 입력된 모든 기능키를 읽어줍니다. 여기에는 키 조합도 포함됩니다. 예를 들면 컨트롤 키와 다른 글자를 함께 누르는 것입니다.

입력중 철자 오류 경고음 재생

이 옵션이 선택되면 입력된 단어에 철자 오류가 있을때 버저음이 재생됩니다. 본 옵션은 문서 서식 설정 대화상자내 철자 오류 알림 옵션이 활성되어 있을때에만 나타납니다.

다른 프로그램에서의 키보드 입력 처리

이 옵션은 화상 키보드나 음성 인식 소프트웨어와 같은 프로그램 사용중 입력된 명령을 NVDA가 처리할지를 설정합니다. 기본으로 이 옵션은 선택되어 있으나 만약 UniKey 입력 소프트웨어를 이용하여 베트남어를 입력하는 사용자들은 오타 방지를 위해 본 옵션을 해제해 주십시오.

12.1.9. 마우스 설정 (NVDA+control+m)

마우스 설정 페널을 통해 마우스 추적, 좌표 비프음 알림 등과 같은 마우스 사용 관련 옵션을 설정할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

마우스 모양 변경 알림

이 체크박스가 체크된 경우, NVDA는 마우스 모양이 변할 때마다 마우스 포인터의 형태를 말해 줄 것입니다. 윈도우에서 마우스 포인터는 어떤 것이 편집 가능 할 때 혹은 어떤 것이 로딩 중일 때처럼 특정한 정보를 전달하기 위하여 형태를 변경합니다.

마우스 위치 추적 사용

단축키: NVDA+m

이 옵션을 선택하면, NVDA는 마우스의 움직임에 따라서 마우스 포인터가 위치한 텍스트를 읽게 됩니다. 이 기능은 화면 속의 어떤 것을 찾을 때에 객체 탐색 방식 보다 물리적인 마우스 동작 방식을 더 유리하게 만듭니다.

마우스 추적 시에 텍스트 단위 설정 (Text unit resolution)

NVDA가 마우스의 움직임에 따라서 텍스트를 읽도록 설정된 경우 이 옵션에서는 정확하게 얼마의 텍스트를 읽을지를 선택할 수 있습니다. 옵션은 글자 단위, 단어 단위, 줄 단위, 문단 단위입니다.

텍스트 단위 설정을 어느곳에서나 토글하려면 단춧키 설정 대화상자를 통해 토글 명령을 설정하세요.

마우스로 객체 진입 시 역할 알림

만약 이 체크박스가 체크되었다면, NVDA는 마우스가 객체 내에서 움직임에 따라 객체의 역할(타입)을 알려 줄 것입니다.

마우스 이동시 경고음으로 좌표 알림 (Play audio coordinates when mouse moves)

이 체크박스를 체크하면 NVDA는 마우스의 움직임에 따라 비프음을 출력합니다. 이때 사용자는 화면의 디멘션과 관련하여 마우스가 어디에 있는지를 알 수 있습니다. 비프음의 고저는 화면상 마우스의 위치(위 또는 아래)를 가리키며 마우스가 화면의 왼쪽 또는 오른쪽에 있느냐에 따라 좌우 볼륨이 변경됩니다(스테레오 스피커나 해드폰 사용시).

밝기에 따른 비프음 크기 제어 (Brightness controls audio coordinates volume)

"마우스 이동에 따른 비프음 출력" 체크박스가 체크되어 있고 이 체크박스도 체크되었다면, 마우스 포인터 아래의 화면이 얼마나 밝은지에 따라서 비프음의 크기를 제어합니다. 이 체크박스는 초기값으로 해제되어 있습니다.

Ignore mouse input from other applications

This option allows the user to ignore mouse events (including mouse movement and button presses) generated by other applications such as TeamViewer and other remote control software. This option is unchecked by default. If you check this option and you have the "Enable mouse tracking" option enabled, NVDA will not announce what is under the mouse if the mouse is moved by another application.

12.1.10. 터치 기능 설정

터치 기능 설정 페널을 통해 NVDA의 터치 기능을 설정할 수 있습니다(윈도우 8 이상이 설치된 터치 기능이 장착된 컴퓨터에서만 접근 가능). 본 설정 페널은 다음 옵션을 포함합니다:

Enable touch interaction support

This checkbox enables NVDA's touch interaction support. If enabled, you can use your fingers to navigate and interact with items on screen using a touchscreen device. If disabled, touchscreen support will be disabled as though NVDA is not running. This setting can also be toggled using NVDA+control+alt+t.

터치 타이핑 모드

이 체크박스를 통해 터치 키보드 입력 방식을 설정할 수 있습니다. 터치 키보드에서 원하는 키를 찾은 후 손가락을 떼면서 곧바로 선택된 키를 입력하려면 이 체크박스를 체크합니다. 이와 반대로 두번 태핑하여 키를 입력하려면 본 체크박스를 해제합니다.

12.1.11. 리뷰 커서

리뷰 커서 페널을 통해 리뷰 커서 사용 관련 옵션을 설정할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

시스템 포커스 따라가기

단축키: NVDA+7

이 옵션을 선택하면, 시스템 포커스가 이동하더라도 리뷰 커서는 항상 현재 시스템 포커스와 동일한 객체에 위치하게 됩니다.

시스템 케럿 따라가기

단축키: NVDA+6

이 옵션을 선택하면, 시스템 케럿이 이동하더라도 리뷰 커서는 시스템 캐럿의 위치로 자동적으로 이동합니다.

마우스 커서 따라가기

이 옵션을 선택하면 리뷰커서는 마우스의 움직임에 따라서 마우스를 따라갈 것입니다.

심플 리뷰 모드

이 옵션을 선택하면, NVDA는 보이지 않는 객체와 레이아웃용 객체 등 사용자에게 관심을 끌지 못하는 객체들을 배제하기 위하여 탐색 가능한 객체들의 서열을 추출할 것입니다.

어느 곳에서나 심플 리뷰 모드를 토글하려면 단춧키 설정 대화상자를 통해 핫키를 설정하세요.

12.1.12. 화면 객체 표시 (NVDA+control+o)

특정 화면 객체에 대한 정보(설명, 위치 등) 출력을 설정하려면 화면 객체 표시 페널에 나열된 옵션을 설정합니다. 본 설정 페널은 다음 옵션을 포함합니다:

도구 설명 알림

이 체크박스가 체크되면, NVDA는 도구 설명(툴팁)이 나타날 때마다 그것을 읽습니다. 많은 윈도우의 컨트롤들은 마우스 포인터를 컨트롤로 가져가면 (툴팁이나) 작은 메시지를 보여줍니다.

도움말 표시란 알림

이 체크박스가 체크되면 NVDA는 도움말 표시란(풍선 도움말 또는 토스트)이 나타날 때마다 그것을 읽습니다.

  • 풍선 도움말은 툴팁과 비슷합니다. 그러나 툴팁보다 크기가 더 크고 같은 시스템 이벤트들과 연결되어 있습니다. 예를 들면, 네트워크가 끊어졌다거나 윈도우즈 보안 이슈에 관하여 경고하기 등입니다.
  • 토스트는 윈도우 8 이상에서 나타나는 알림창으로 여러 이벤트 관련 알림을 보여줍니다(예: 업데이트 다운로드, 이메일 도착 등).

객체 핫키 알림

이 체크박스가 체크되면, 특정한 객체나 컨트롤이 핫키를 지원하는 경우 NVDA는 그 객체나 컨트롤과 연결된 핫키를 함께 알려줍니다. 예를 들면 메뉴바의 파일 메뉴는 알트+F라는 핫키를 가지기도 합니다.

객체 위치정보 알림

이 옵션에서는 포커스로 객체 이동을 하거나 객체 탐색을 할 때 알려지는 객체의 위치 정보를(예를 들면 1 / 4) 일게 할 것인지 아닌지를 선택합니다.

객체 위치 없을때 위치 추정하기

객체 위치정보 알림을 선택하였어도 특정 객체에서 이 기능이 제대로 작동하지 않을 때 이 옵션은 NVDA가 객체의 위치정보를 추정할 수 있도록 합니다.

이 옵션을 선택할 때, NVDA는 메뉴나 툴바 등 더 많은 컨트롤들의 위치정보를 알려 줄 것입니다. 그러나 이 정보는 약간 부정확할 수도 있습니다.

객체 설명 알림

객체에 딸려 있는 설명이 필요없다면 이 체크박스를 해제하세요.

진행율 출력

단축키: NVDA+u

이 옵션은 NVDA가 진행율 변경을 어떻게 알려 줄 것인지를 제어합니다.

여기에는 다음 옵션이 있습니다.

  • 끔: 진행율이 변화해도 읽지 않습니다.
  • 말하기: 이 옵션을 선택하면 NVDA는 진행율을 백분율로 읽습니다. 진행율이 변경될 때마다 NVDA는 새로운 값을 읽을 것입니다.
  • 비프음: 이 옵션에서는 진행율이 변경될 때마다 NVDA가 비프음을 출력하도록 선택할 수 있습니다. 비프음이 높아지면 높아질수록 진행율은 완료에 가까워집니다.
  • 비프음과 말하기: 이 옵션에서는 NVDA가 진행율의 변화를 비프음으로 출력하는 동시에 말하도록 선택할 수 있습니다.

백그라운드 진행율 알림

이 옵션을 체크하면 진행율 표시줄이 실제로 포그라운드되지 않았다 하더라도 NVDA는 진행율 변경 알림을 유지합니다. 진행율 표시줄이 포함된 윈도우를 최소화하거나 백그라운드로 전환한 경우에도 NVDA는 진행율을 추적하기 때문에 그동안 다른 일을 할 수 있습니다.

자동 변경된 컨텐츠 알림

단축키: NVDA+5

터미널과 채팅 프로그램의 history 제어 등 특정 객체에서 새로운 컨텐츠 읽기 온/오프를 토글합니다.

검색 결과 나타날때 사운드로 알림

검색 결과가 나타날때 이것을 사운드로 알려줄지를 토글합니다. 검색 결과, 또는 자동 권장(auto-suggestion)이란 특정 편집창 또는 문서에서 텍스트 검색시 나타나는 결과 목록을 말합니다. 예를 들어 윈도우 비스타 이상에서 시작 메뉴의 검색 편집창에 텍스트를 입력시 윈도우는 입력된 텍스트를 기반으로 검색 결과 목록을 보여줍니다. 특히 윈도우 10이상에서 여러 앱 사용시 특정 편집창에서 텍스트를 입력시 NVDA는 검색 결과가 나타났다고 알려줄 수 있습니다. 검색창을 빠져나갈때 검색 결과 목록은 사라지며 이때 여러 편집창에서 이것을 사운드로 알려줄 수 있습니다.

12.1.13. 입력 구성 알림

입력 구성 알림 페널을 통해 텍스트 서비스 및 IME를 이용한 아시아 언어 입력을 NVDA가 어떻게 처리할 것인지 설정할 수 있습니다. 주의: 각 입력 방식에 따라 설정값을 조정하시기 바랍니다.

후보 목록 자동으로 알림

이 설정은 후보자 목록이 나타났을때에 목록 전체를 읽을것인지를 설정합니다. 기본으로 이 설정은 선택되어 있습니다. 일반적으로 그림형 문자(신 중국어 등 문자)를 입력할때 후보자 번호 및 목록 전체를 읽음으로 입력하고자 하는 후보자 숫자가 나타나면 숫자를 사용하여 바로 입력이 가능합니다. 그러나 한국어처럼 음성 문자(여러 단어의 발음이 같은 경우)를 입력할 경우 같은 발음이 반복되어 출력될 수 있음으로 이 설정을 해제하시기 바랍니다.

선택된 후보 알림

이 설정은 후보자 목록에서 방향키로 후보자 목록을 탐색할때 선택된 후보자를 알릴지를 설정합니다 (기본으로 선택되어 있습니다). 한자와 같이 여러 후보자간 이동하면서 후보자를 선택하는 경우 이 옵션을 선택하시기 바랍니다. 만약 이 옵션을 해제해도 리뷰 커서는 항상 후보자 목록에 위치하게 됩니다.

후보 출력 시 글자와 풀이도 알림

이 설정은 후보자 목록 출력시 각 후보자별 풀이를 읽을것인지를 설정합니다. 만약 한자를 입력하는 경우 후보자 자동 알림을 선택하셨다면 이 옵션을 선택하여 풀이를 들으면서 입력하시기 바랍니다.

읽기 문자열 변환값 알림

이 설정은 글자 입력후 일기 문자열 변환값을 알려줄것인지를 설정합니다.

입력 문자 변환값 알림

이 설정은 아시아 글자 입력시 현재 글자 문자 변경값을 읽을것인지를 설정합니다. 기본으로 이 설정은 선택되어 있습니다.

12.1.14. 브라우즈 모드 (NVDA+control+b)

브라우즈 모드 페널을 통해 웹페이지와 같은 문서를 읽을때의 NVDA 기능을 설정할 수 있습니다. 본 설정 페널은 다음 옵션을 포함합니다:

줄당 최대 글자수

이 필드에서는 브라우즈 모드에서 한 줄의 최대 길이를 (글자 단위로) 설정합니다.

페이지당 최대 줄수

이 필드에서는 브라우즈 모드에서 페이지업 키나 페이지다운 키를 누를 때 이동하고자 하는 줄 수를 설정합니다.

화면 레이아웃 사용

단축키: NVDA+v

이 옵션에서는 브라우즈 모드에서 내용이 표시될 때 링크와 기타 필드들과 같은 내용을 별도의 줄로 표시할 것인지 아니면 눈에 보이는 그대로 텍스트의 흐름 속에 그것들을 유지할 것인지를 지정할 수 있습니다. 이 옵션이 유효하면 그것들은 눈에 보이는 그대로 유지될 것입니다. 반대로 이 옵션이 해제되었다면, 그 필드들은 별도의 줄로 표시될 것입니다.

페이지가 로드되면 브라우즈 모드 활성화

이 체크박스는 페이지가 로드된 후 자동으로 브라우즈 모드를 활성화할지 토글합니다. 이 옵션을 사용하지 않을 경우 브라우즈 모드를 지원하는 앱 또는 문서에서 브라우즈 모드를 수동으로 토글할 수 있습니다. 브라우즈 모드를 지원하는 앱 목록은 브라우즈 모드 섹션에서 확인할 수 있습니다. 본 설정은 마이크로소프트 워드와 같이 브라우즈 모드를 선택적으로 사용할 때에는 적용되지 않습니다. 이 옵션은 초기값으로 유효하게 설정됩니다.

페이지 로딩 시 자동으로 읽기

이 체크박스는 브라우즈 모드에서 페이지가 로드된 후 그 페이지를 자동으로 읽을 것인지를 토글합니다. 이 옵션은 초기값으로 유효하게 설정됩니다.

레이아웃 테이블 포함

이 옵션은 순수하게 시각적으로 표시된 구조 표(레이아웃 테이블)를 NVDA가 어떻게 처리할지를 제어합니다. 이 옵션이 선택되면 사이트가 구조 표를 사용할때 첫 글자를 사용하여 구조 표로 움직일 수 있습니다. 또한 NvDA는 그 구조 표의 내용을 문서 서식 설정에서 설정된 설정을 사용하여 읽습니다. 그러나 이 옵션이 해제되면 첫 글자로 탐색시 구조 표로 이동할 수 없으며 또한 구조 표 자체도 인식하지 않습니다 (단 구조 표의 내용은 일반 텍스트처럼 읽을 수 있음). 본 옵션은 기본으로 해제되어 있습니다.

본 옵션을 어느곳에서나 토글하려면 단춧키 설정 대화상자를 통해 토글 명령을 설정하세요.

링크나 헤딩 같은 요소 읽기 설정하기

링크, 헤딩, 표처럼 탐색할 때 읽게 되는 요소를 설정하려면 문서 서식 설정 페널에서 옵션을 참조하세요.

포커스 이동 시 자동 포커스 모드 전환

이 옵션은 포커스가 변경될 때 포커스 모드로 전환될 수 있게 합니다. 예를 들면 웹페이지에서 탭을 눌러 어떤 폼에 들어갈 때, 이 옵션에 체크되어 있다면 자동으로 포커스 모드로 전환될 것입니다.

캐럿 이동 시 자동 포커스 모드 전환

이 옵션이 체크되면 NVDA는 방향키를 사용할 때 포커스 모드로 진입하거나 나갈 수 있습니다. 예를 들면 웹페이지에서 방향키로 편집창에 들어 갈 경우 NVDA는 자동으로 포커스 모드로 진입할 것입니다. 방향키로 편집창을 빠져 나올 경우 NVDA는 브라우즈 모드로 되돌아 올 것입니다.

포커스와 브라우즈 모드 전환시 경고음 재생

만약 이 옵션을 선택하면 NVDA는 브라우즈 모드와 포커스 모드가 서로 전환될 때 특수한 소리를 출력하여 알려줍니다. 그러나 모드의 전환을 말해주지는 않습니다.

브라우즈 모드외 명령을 문서가 처리하지 않도록 함

본 설정은 현재 포커스된 문서에서 브라우즈 모드외 명령(즉 NvDA 명령이 아닌 키들)을 문서가 처리할지(문서에 보낼지)를 묻습니다(기본으로 선택되어 있음). 예를 들어 본 옵션을 선택한 후 문서에서 J 키를 눌렀을때 브라우즈 모드 명령이 아니거나 또는 문서가 본 키를 사용한다 할찌라도 J 키는 문서에 보내지지 않습니다. 브라우즈 모드 명령이 문서에 보내지지 않을때 윈도우에서 기본음이 재생됩니다.

자동으로 포커스 가능한 요소에 시스템 포커스 이동

단축키: NVDA+8

본 옵션을 통해 브라우즈 모드로 탐색시 포커스 가능한 요소(링크, 폼 필드 등)에 시스템 포커스를 옮길지를 토글합니다(기본으로 선택되지 않음). 본 옵션을 사용하지 않는 경우 포커스가 가능한 요소를 브라우즈 모드로 접근시 시스템 포커스는 움직이지 않습니다. 이때 브라우즈 모드 탐색중 NVDA의 반응 속도가 빨라질 수 있습니다. 시스템 포커스는 특정 요소를 활성화할때(버튼 누르기, 체크박스 선택 등) 옮겨지게 됩니다. 본 옵션을 사용할시 특정 웹사이트 지원은 향상될 수 있으나 반응 속도 저하 등의 문제가 있을 수 있습니다.

WordPad처럼 서식이 중요한 애플리케이션에서 문서를 읽으면서 검토하는 동안에는 이 옵션을 선택하세요.

12.2. 그 외 설정

NVDA 메뉴의 설정 하위메뉴에는 NVDA 설정 대화상자 외 다른 설정 항목들도 나열되어 있습니다. 이 항목들을 통해 아래에 나열된 설정 대화상자들을 호출할 수 있습니다.

12.2.1. 음성 발음 사전

설정 메뉴에 있는 음성 발음 사전 메뉴는 NVDA가 특정한 단어나 구를 어떻게 읽을 것인지를 관리하는 대화상자를 포함합니다. 현재 세가지 서로다른 타입의 음성 발음 사전이 있습니다. 음성 발음 사전은 다음과 같습니다.

  • 기본 발음 사전: 이 사전의 규칙들은 NVDA의 모든 발음에 적용됩니다.
  • 보이스 발음 사전: 이 사전의 규칙들은 현재 사용 중인 음성 엔진의 보이스 발음에 적용됩니다.
  • 임시 발음 사전: 이 사전의 규칙들은 NVDA의 모든 발음에 적용됩니다. 그러나 단지 현재 세션에만 해당합니다. 이 규칙들은 임시적이며 NVDA가 재시작할 때 없어집니다.

어느곳에서나 위 발음 사전들을 열려면 단춧키 설정 대화상자에서 각 사전별 핫키를 설정하시기 바랍니다.

모든 발음 사전 대화상자는 발음을 처리하는 과정에서 사용될 규칙의 목록을 포함합니다. 이 대화상자는 추가, 수정, 삭제 버튼도 포함합니다.

발음 사전에 새로운 규칙을 추가하려면 추가 버튼을 누르고 나타난 대화상자에 있는 필드에 정보를 채우고, Ok를 누르세요. 그러면 새로운 규칙을 규칙 목록에서 볼 수 있을 것입니다. 그러나 규칙을 확실하게 저장하려면 규칙을 추가/수정한 후 반드시 Ok(확인)를 선택하십시오.

NVDA 음성 발음 사전의 규칙은 한 문자열을 다른 문자열로 바꾸어 읽도록 해 줍니다. 간단한 예를 들면, 여러분은 NVDA가 "새"라고 읽어야 할 "새"라는 단어를 매번 "개구리"라고 일게 하고 싶습니다. 이것을 가장 쉽게 처리하는 방법은 추가 규칙 대화상자에서 단어 "새"를 패턴 필드에 입력하고 단어 "개구리"를 교체 필드에 입력하는 것입니다. 주석 필드에 규칙에 대한 설명을 입력할 수도 있습니다. )예: 새를 개구리로 변경)

그러나 NVDA의 음성 발음 사전은 간단한 단어 교체 보다 훨씬 더 강력합니다. 추가 규칙 대화상자에 있는 "대소문자 구별" 체크박스를 통해 추가하고자 하는 규칙에 대소문자 구분 여부를 지정할 수 있습니다. (NVDA는 초기값으로 대소문자의 구별을 무시합니다.)

마지막으로 적용 방식 라디오 버튼을 통해 페턴을 무제한, 단어 전체 또는 정규 표현으로 처리할지를 지정할 수 있습니다. 페턴 적용 방식을 단어 전체로 지정하면 페턴 앞뒤에 아무런 글자(예: 언더바(_) 또는 숫자 및 글자)가 없는 경우(즉 페턴이 한 단어의 일부가 아닐때) 새로 추가된 규칙이 적용됩니다. 예를 들어 위에서 언급했던 엔트리("개구리"를 "새"로 읽도록 함)를 단어 전체에만 적용하도록 했다면 "왕개구리"나 "파랑새"는 적용 페턴에서 제외됩니다.

정규 표현식은 특수 기호를 포함하는 패턴인데, 그것은 몇몇 예시처럼 한번에 한 글자 이상에서 일치하거나 단지 숫자만이 일치하거나 단지 글자만 일치하거나 할 수 있습니다. 정규 표현식에 관해 이 사용자 안내서는 충분치 못합니다. 그러나 상세한 정보를 제공하는 설명서들이 웹사이트에 많이 있습니다.

12.2.2. 구두점/기호 발음 설정

이 대화상자에서는 구두점 및 다른 기호들의 읽기 레벨 뿐만 아니라 그것들이 발음되는 방식을 변경할 수 있습니다.

제목 표시줄에는 현재 사용중인 언어가 표시됩니다. 특히 본 대화상자에 나열된 기호명은 현재 설정된 보이스 또는 인터페이스 언어를 존중합니다(즉 보이스 설정에 있는 "음성 언어 데이타 사용"을 선택하면 현재 음성 언어로 기호가 출력됩니다).

기호의 발음을 변경하려면 먼저 바꾸고자 하는 기호를 기호 목록에서 선택하세요.

  • 변환값 필드(replacement field)에서는 이 기호 대신에 발음될 텍스트를 변경할 수 있습니다.
  • 레벨 필드를 이용하여 이 기호가 발음될 가장 낮은 기호 레벨을 조절할 수 있습니다.
  • 기호 자체 처리 필드를 사용하여 음성 엔진에서 기호를 처리할지를 설정할 수 있습니다. 즉 기호의 변환값 대신 기호 자체를 음성 엔진으로 보내어 띄어 읽기(예: 쉼표를 만나면 띄어 읽기)나 억양 변경을 조절하고자 할때 유용합니다. 다음 세 옵션중 선택하세요:
    • 처리 않함: 기호 자체를 음성 엔진에 보내지 않습니다.
    • 처리함: 기호 자체를 음성 엔진에 보냅니다.
    • 레벨 이하일때 처리함: 현재 NVDA의 기호 발음 설정이 선택된 기호 발음 레벨 이하일때 기호 자체를 음성 엔진에 보냅니다. 예를 들어 특정 기호 발음 레벨에서는 변환값을 알리고 그 이상의 레벨에서는 띄어 읽도록 할 수 있습니다.

본 대화상자에서 추가 버튼을 눌러 새 기호를 추가할 수 있습니다. 추가 대화상자가 열리면 추가하고자 하는 기호를 입력한 후 확인(OK)를 선택합니다. 새 기호에 대해서도 다른 기호들과 마찬가지로 여러 필드를 통해 변환값 및 여러 옵션을 설정하십시오.

사용자가 추가한 기호를 삭제하려면 삭제 버튼을 선택하십시오.

작업을 마쳤다면, OK 버튼을 눌러 변경 사항을 저장하거나 Cancel 버튼을 눌러 작업을 취소하십시오.

12.2.3. 단축키 설정

이 대화상자에서는 NVDA에서 사용되는 명령 단축키 (키보드 명령, 점자디스플레이 키 등)를 조정할 수 있습니다.

본 대화상자에 나열된 명령들은 단축키 설정을 호출하기전 사용중이던 명령들입니다. 예를 들어 브라우즈 모드를 사용하던중 단축키 설정을 호출시 브라우즈 모드 관련 명령이 나열되며 이중 특정 명령에 대한 단축키를 조정할 수 있습니다.

각 NVDA 명령은 명령 용도로 구분되어 트리 형태로 표시되며 명령 설명 항목을 확장하여 그 명령에 할당된 단축키를 확인할 수 있습니다. 또한 검색 편집창을 이용하여 원하는 명령을 검색할 수 있습니다.

특정 명령에 단축키를 지정하려면 명령을 선택한 후 추가 버튼을 누릅니다. 이 후 할당하고자 하는 단축키(키보드 명령, 점자 명령 등) 자체를 입력합니다. 가끔 지정하고자 하는 단축키가 여러 레이아웃 형태로 존재할 수 있습니다. 예를 들어 새 키보드 단축키를 모든 레이아웃 또는 특정 레이아웃(데스크탑, 노트북 등)에 적용할 수 있습니다. 이때 단축키 적용 메뉴가 호출되며 이 메뉴에서 단축키 지정 방식을 선택할 수 있습니다.

특정 명령에서 단축키를 제거하려면 명령 단축키를 선택한 후 제거 버튼을 누르십시오.

설정을 마친 후 확인(OK) 버튼을 눌러 설정을 저장하거나 취소(Cancel) 버튼을 눌러 이전 저장된 값으로 환원하십시오.

12.3. 환경 설정 저장 및 환원

기본적으로 NVDA는 종료 시에 환경 설정을 자동으로 저장합니다. 이 설정은 NVDA의 일반 설정에서 변경할 수 있습니다. 언제든지 수동으로 환경 설정을 저장하려면 NVDA 메뉴에서 "환경설정 저장"을 선택하세요.

만약 환경 설정에 잘못이 있어서 기존에 저장된 환경 설정으로 되돌아 갈 필요가 있다면, NVDA메뉴에서 "저장된 환경 설정으로 환원"을 선택하세요. 또한 NVDA 메뉴에서 "기본 환경설정으로 초기화"를 실행하여 최초 설치 당시의 환경 설정으로 초기화할 수 있습니다.

다음의 NVDA 단축키도 유용합니다.

이름 데스크탑 키 노트북 키 설명
환경 설정 저장하기 NVDA+control+c NVDA+control+c NVDA를 종료해도 환경 설정 값을 잃지 않도록 현재 환경 설정을 저장합니다.
기존환경 설정으로 환원 NVDA+control+r NVDA+control+r 한번 누르면 최신 저장된 환경설정으로 돌아가고 세번 누르면 설정을 NVDA 설치때의 설정으로 되돌립니다.

12.4. 환경 설정 프로필

사용자는 상황에 따라 각기 다른 환경 설정을 사용할 수 있습니다. 예를 들어 편집시와 읽기시 각기 다른 설정을 사용할 수 있습니다. 이러한 상황별 NVDA 설정을 묶은것을 환경 설정 프로필이라고 합니다.

특정 프로필이 적용되었을때 사용자가 변경하는 설정값은 현재 사용중인 프로필에 적용됩니다. NvDA의 일반 설정을 제외한 나머지 설정값을 프로필에 적용시킬 수 있습니다(일반 설정은 NvDA 전체 환경을 총괄함으로 제외).

프로필이 지정되면 사용자는 그 프로필을 수동으로 적용할 수 있습니다. 또한 프로그램 실행과 같은 엑션(트리거)에 특정 프로필을 적용시킬수도 있습니다.

12.4.1. 프로필 관리

사용자는 NVDA 메뉴에 있는 환경 설정 프로필 옵션을 통해 프로필을 관리할 수 있습니다. 또한 밑 단축키를 사용하여 프로필 설정에 접근할 수 있습니다:

  • NVDA+control+p: 환경 설정 프로필 대화상자에 접근합니다.

환경 설정 프로필 대화상자를 열면 현재 편집중인 프로필(선택되어 있음)을 포함한 프로필 목록이 나타납니다. 또한 현재 적용된 프로필에 대한 정보(편집 여부, 수동 또는 자동 적용)도 보여줍니다.

선택된 프로필을 삭제하거나 이름을 변경하고자 한다면 제거 또는 이름 변경 버튼을 선택하십시오.

닫기 버튼을 누르면 본 대화상자가 닫혀집니다.

12.4.2. 프로필 만들기

프로필을 생성하려면 새 프로필 버튼을 선택하십시오.

새 프로필 대화상자에서 생성하고자 하는 프로필의 이름을 입력하십시오. 또한 프로필 적용 시기도 설정할 수 있습니다. 새 프로필을 수동으로 적용하려면 수동 적용(기본값)을 어떤 엑션(트리거)에 의해 자동으로 적용하려면 적용 엑션을 선택하십시오. 만약 프로필 이름을 입력하지 않은 상태에서 자동(트리거) 적용을 선택하면 사용자 편의상 선택된 엑션값이 프로필 이름으로 입력됩니다. 프로필 적용 엑션(트리거)에 대해서는 밑 섹션을 참고하세요.

사용자가 등록한 프로필을 적용/편집하려면 확인(OK)를 누르세요.

12.4.3. 프로필 수동 적용

프로필을 수동으로 적용하려면 프로필을 선택한 후 수동 적용 버튼을 선택합니다. 사용자가 선택한 프로필을 수동으로 적용한 후에도 상황별 엑션에 할당된 프로필이 자동으로 적용될 수 있습니다. 그러나 수동으로 적용된 프로필의 설정값이 효력을 가지며 설정이 변경될때 수동 프로필에 저장됩니다. 예를 들어 현재 실행중인 프로그램에서 어느 프로필이 적용되도록 설정했다고 합시다. 만약 링크 알림 설정이 수동 프로필에서는 해제되었고 현재 적용된 프로필에서는 선택되었다면 그 프로그램 사용시에만 링크가 알려집니다. 이와 같이 자동으로 적용된 프로필에서 음성을 변경하였다면 자동으로 적용된 프로필에서 새 음성이 출력됩니다. 수동으로 적용된 프로필을 해제하려면 프로필을 선택한 후 적용 해제 버튼을 선택하십시오.

12.4.4. 프로필 적용 시기 및 엑션

프로필 적용 시기 버튼을 선택하여 상황에 따라 어느 프로필을 적용할 것인지를 설정할 수 있습니다.

적용 시기 목록에서 아래 항목 중 하나를 선택하십시오:

  • 프로그램 사용시: 특정 프로그램을 사용시 적용됩니다.
  • 모두 읽기시: 모두 읽기를 사용시 적용됩니다.

특정 상황에서 사용될 프로필을 지정하려면 적용 시기를 선택한 후 그 시기에 사용할 프로필을 선택합니다. 만약 어떠한 프로필도 사용하지 않고싶다면 일반 설정을 선택하십시오.

닫기 버튼을 눌러 프로필 설정 대화상자로 빠져나가십시오.

12.4.5. 프로필 편집

특정 프로필을 수동으로 적용하면 사용자가 변경하는 설정값들은 그 프로필에 저장되게 됩니다. 그러나 만약 엑션에 할당된 프로필이 적용되었을때에는 자동으로 적용된 프로필에 묶여있는 설정값을 변경할 수 있습니다. 예를 들어 메모장 사용 관련 프로필을 지정하였다면 메모장을 사용할 때 그 프로필에 할당된 설정값을 변경할 수 있습니다. 끝으로 수동 또는 자동으로 적용된 프로필이 없다면 사용자가 변경하는 설정값들은 사용자 기본 환경에 저장됩니다.

만약 편집하고자 하는 프로필을 모두 읽기시에 적용하도록 설정하였다면 그 프로필을 수동으로 적용한 후 편집이 가능합니다.

12.4.6. 프로필 엑션 적용 임시 해제

가끔 엑션별 자동 적용 설정을 해제할 필요가 있습니다. 예를 들어 수동으로 적용한 프로필을 편집할때나 자동으로 적용되는 프로필과 일반 설정이 중복할 경우입니다. 이때 프로필 설정에 있는 프로필 엑션 적용 임시 해제 박스를 선택해 주십시오.

어디서나 프로필 적용 임시 해제를 토글하려면 단춧키 설정 대화상자에서 핫키를 설정하세요.

12.5. 환경설정 파일 저장 위치

NVDA 휴대용 버전은 설정값들과 추가 기능들을 NVDA 디렉토리 안에 있는 userConfig 디렉토리에 저장합니다.

NVDA 설치 버전은 설정값들과 추가 기능들을 각 사용자 폴더 안에 있는 NVDA 디렉토리에 저장합니다. 즉 시스템의 각 사용자가 자신만의 NVDA 환경설정을 가질 수 있음을 의미합니다. NVDA 설치 버전용 설정 디렉토리를 찾으려면 윈도우즈 시작 메뉴에서 프로그램 -> NVDA -> "NVDA 사용자 환경 폴더 탐색" 항목을 선택하세요.

윈도우 로그온 화면이나 사용자 계정 컨트롤(UAC) 화면에서 NVDA를 실행하는 설정은 NVDA가 설치된 디렉토리 안의 systemConfig 디렉토리에 저장됩니다. 대체로 이 환경설정은 건드릴 필요가 없습니다. 윈도우 로그온/UAC 화면에서 NVDA가 실행되도록 환경 설정을 변경하려면 윈도우에 로그온되어 있는 동안 사용자가 원하는 방식으로 NVDA를 설정한 후 일반 설정 대화상자에 있는 버튼을 사용하여 사용자 환경을 시스템 환경으로 복사하면 됩니다.

13. 추가 도구

13.1. 로그 보기

NVDA 메뉴 -> 도구에 있는 로그 보기에서는 최근 NVDA의 시작부터 현재까지 등록된 모든 로그 기록 내용을 확인할 수 있습니다.

로그 내용을 읽는 것 뿐만 아니라 로그 파일의 복사본을 저장할 수도 있고, 로그 보기가 실행된 이후 가장 최근의 로그 기록 출력을 보이도록 새로고침을 할 수 있습니다. 이 기능들은 로그 보기의 로그 메뉴에서 사용 가능합니다.

13.2. 음성 출력 뷰어

정안인 소프트웨어 개발자 혹은 NVDA를 정안인 관객들에게 설명하는 사람들을 위하여 NVDA의 음성 출력 내용을 플로팅(floating) 윈도우로 보여줄 수 있습니다.

음성 출력 뷰어를 사용하려면 NVDA 메뉴 -> 도구 -> "음성 출력 뷰어" 메뉴 항목을 체크하세요. 이 항목을 체크 해제하면 음성 출력 뷰어가 정지됩니다.

또한 NVDA 시작시 음성 출력 뷰어가 열리도록 설정할 수 있습니다. 본 옵션은 음성뷰어 창에 있는 "NVDA 시작시 음성 출력 뷰어 보이기" 체크박스를 통해 조절할 수 있습니다. 뷰어 창이 열릴때마다 이전에 사용하던 창 위치가 복원됩니다.

음성 출력 뷰어 사용시 NVDA는 최신 음성 출력 내용을 음성 뷰어 윈도우에 띄웁니다. 음성출력 뷰어 내부에서 클릭하거나 포커스하면 NVDA는 기존의 내용을 쉽게 선택하거나 복사할 수 있도록 일시적으로 텍스트 업데이트를 중단합니다.

어느곳에서나 음성 출력 뷰어를 토글하려면 단춧키 설정 대화상자에서 음성 뷰어 토글 핫키를 설정하시기 바랍니다.

13.3. 점자 출력 뷰어

For sighted software developers or people demoing NVDA to sighted audiences, a floating window is available that allows you to view braille output, and the text equivalent for each braille character. The braille viewer can be used at the same time as a physical braille display, it will match the number of cells on the physical device. While the braille viewer is enabled, it constantly updates to show you the braille that would be displayed on a physical braille display.

To enable the braille viewer, check the "Braille Viewer" menu item under Tools in the NVDA menu. Uncheck the menu item to disable it.

Physical braille displays typically have buttons to scroll forwards or backwards, to enable scrolling with the braille viewer tool use the Input Gestures dialog to assign keyboard shortcuts which "Scrolls the braille display back" and "Scrolls the braille display forward"

The braille viewer window contains a check box labelled "Show braille viewer on startup". If this is checked, the braille viewer will open when NVDA is started. The braille viewer window will always attempt to re-open with the same dimensions and location as when it was closed.

13.4. 파이썬 콘솔

NVDA 메뉴 -> 도구에 있는 NVDA 파이썬 콘솔은 개발 도구로서 디버깅과 NVDA 내부에 대한 일반적인 검사 혹은 애플리케이션에 대한 접근성 계층 구조의 검사에 유용합니다. 자세한 정보는 NVDA 개발 문건을 참조하세요.

13.5. 추가 기능 관리자

NVDA 메뉴 -도구 -> 추가 기능 관리를 선택하면 추가 기능 관리자가 나타납니다. 추가 기능 관리자에서는 NVDA 추가 기능 패키지를 설치, 제거 또는 사용 여부를 설정할 수 있습니다. NvDA 추가 기능은 NvDA 커뮤니티에서 제공하는 패키지로 NVDA 기능 추가 및 변경, 또는 제3자 음성 엔진 및 점자디스플레이 지원 코드를 내장하고 있습니다.

추가 기능 관리자가 열리면 현재 사용자 환경에 설치된 추가 기능 목록이 나타납니다. 여기에서 각 추가 기능의 이름, 실행 여부, 버전 및 저자에 대한 정보를 확인하실 수 있습니다. 선택된 추가 기능에 대한 자세한 정보(세부 설명, URL 등)를 보려면 추가 기능 정보 버튼을 누르시기 바랍니다. 그리고 추가 기능 도움말(추가 기능 개발자가 도움말 정보를 지정한 경우)을 보려면 추가 기능 도움말 버튼을 선택하십시오.

또한 NVDA 추가 기능 모음 사이트에 접속하여 추가 기능을 탐색 및 다운받으실 수 있습니다. 본 사이트에서 추가 기능을 다운받으시려면 추가 기능 탐색 버튼을 선택하십시오. NVDA가 설치되었다면 웹브라우저를 통해 사이트에서 추가 기능을 다운받은 후 곧바로 추가 기능을 설치할 수 있습니다 (밑을 참조 바람). 만약 설치 버전이 아니라면 본 사이트에서 추가 기능 파일을 다운받은 후 밑 방법을 따라 추가 기능을 설치하십시오.

추가 기능을 설치하려면 위 방법으로 추가 기능을 다운받은 후 추가 기능 관리자에서 설치 버튼을 누릅니다. 여기에서 추가 기능 패키지(.nvda.addon 파일)를 여러분의 컴퓨터나 네트워크에서 찾을 수 있습니다. 설치를 시작하려면 "열기" 버튼을 누르십시오.

추가 기능 설치시 NVDA는 정말로 추가 기능을 설치할 것인지를 묻습니다. 추가 기능의 기능은 NVDA 안에서 무제약적으로 자유롭기 때문에 NVDA가 설치 버전인 경우 이론적으로 개인의 데이터 뿐만 아니라 전체 시스템에도 액세스할 수 있습니다. 그럼으로 사용자가 신뢰하는 소스로부터 제공된 추가 기능 설치를 권장합니다. 추가 기능이 설치되면 갓 설치된 추가 기능을 실행시키기 위하여 반드시 NVDA를 재시작해야 합니다. NVDA를 재시작 하기 전까지 "설치"라는 상태가 목록에 있는 그 추가 기능 항목에 보일 것입니다.

추가 기능을 제거하려면 목록에서 삭제할 추가 기능을 선택하고 제거 버튼을 누르세요. NVDA는 정말로 추가기능을 삭제할 것인지 묻습니다. 설치와 마찬가지로 추가 기능을 완전히 삭제하기 위하여 추가 기능 제거 후 NVDA를 반드시 재시작하십시오. 재시작하기 전까지 "제거"라는 상태가 목록에 있는 그 추가기능 항목에 보일 것입니다.

실행중이거나 사용함으로 된 추가 기능을 일시 정지하려면 "사용 않함" 버튼을 누르고 이전에 정지되었거나 사용 않함으로 설정된 추가 기능을 사용하려면 "사용" 버튼을 누르세요. 토글 버튼을 누를때마다 NVDA 재시작후의 추가 기능 사용 여부가 보여집니다. 만약 추가 기능을 사용함으로 토글한 경후 "재시작 후 사용"으로 표시되고 반대로 추가 기능을 일시 정지시 "재시작 후 사용 않함"으로 표시됩니다. 추가 기능 설치 및 제거와 마찬가지로 추가 기능 사용 여부 변경은 NVDA를 재시작하면 적용됩니다.

또한 추가 기능 관리자는 대화상자를 닫기 위하여 닫기 버튼을 가지고 있습니다. 만약 추가 기능을 설치, 제거 또는 사용 여부를 변경하였다면 NVDA는 변경사항이 적용되도록 재시작하기를 원하는지 우선적으로 확인할 것입니다.

과거에 개별 플러그인과 드라이버를 NVDA 사용자 환경설정 디렉토리에 복사하는 것으로 NVDA의 기능 확장은 가능했습니다. 최신 NVDA 버전에서도 이전 방식의 추가 기능을 동작시킬 수 있습니다마는 구형 추가 기능들은 추가 기능 관리자에 등록되지 않습니다. 만약 이 추가 기능을 사용하고자 한다면 환경 설정 폴더에 있는 구형 추가 기능 파일을 삭제한 후 추가 기능 페키지로 제작된 버전 사용을 권장합니다.

어느곳에서나 추가 기능 관리자를 열려면 단춧키 설정 대화상자에서 추가 기능 관리자 핫키를 설정하시기 바랍니다.

13.5.1. 호환되지 않는 추가 기능 관리자

추가 기능 관리자에서 "호환되지 않는 추가 기능" 버튼을 눌러 접근할 수 있는 곳으로 현재 사용중인 NVDA 버전과 호환되지 않는 추가 기능 목록과 호환되지 않는 이유를 보여줍니다. 만약 최신 NVDA 중요 버전과 호환되지 않거나 이전에 존재했으나 지금은 없어진 기능을 사용하는 추가 기능은 호환되지 않는 추가 기능으로 분류됩니다. 호환되지 않는 추가 기능 관리자는 호환 메시지와 현재 NVDA 버전을 알려줍니다. 또한 호환되지 않는 추가 기능 목록도 보여주는데 이 목록은 다음 정보를 포함하고 있습니다:

  1. 추가 기능 이름
  2. 추가 기능 버전
  3. 호환되지 않는 이유

또한 이 관리자에 있는 "추가 기능 정보" 버튼을 통해 추가 기능 정보를 확인할 수 있습니다. 특히 추가 기능 호환에 대해 개발자와 연락시 유용합니다.

13.6. 휴대용 버전 만들기

NVDA가 설치된 경우 이 옵션을 통해 휴대용 NVDA 버전을 만들 수 있습니다. 또한 휴대용 버전에서는 NVDA를 설치할 목적으로 "NVDA 설치"로 나타납니다.

이 대화상자에서 휴대용 버전이 저장될 경로 또는 설치시 설치 경로를 지정합니다.

또한 이 대화상자에서 다음 옵션을 선택할 수 있습니다:

  • 사용자 환경 복사 (설치 버전일 경우 %appdata%\roaming\NVDA 또는 휴대용 버전에 있는 사용자 환경 파일 포함 (추가 기능 및 다른 모듈 포함))
  • 휴대용 버전 만든후/설치 후 자동으로 NVDA 시작 (휴대용 버전을 생성하거나 NVDA를 설치한 후 자동으로 NVDA를 실행함)

13.7. COM Registration 재설정

Installing and uninstalling programs on a computer can, in certain cases, cause COM DLL files to get unregistered. As COM Interfaces such as IAccessible depend on correct COM DLL registrations, issues can appear in case the correct registration is missing.

This can happen i.e. after installing and uninstalling Adobe Reader, Math Player and other programs.

The missing registration can cause issues in browsers, desktop apps, task bar and other interfaces.

Specifically, following issues can be solved by running this tool:

  • NVDA reports "unknown" when navigating in browsers such as Firefox, Thunderbird etc.
  • NVDA fails to switch between focus mode and browse mode
  • NVDA is very slow when navigating in browsers while using browse mode
  • And possibly other issues.

13.8. 플러그인 재등록

이 옵션을 선택하면 NVDA 재시작 없이도 애플리케이션 모듈과 전역 플러그인을 다시 불러옵니다. 이 옵션은 개발자에게 유용합니다.

14. 지원하는 음성 엔진

이 섹션은 NVDA가 지원하는 음성 엔진에 관한 정보를 포함하고 있습니다. NVDA에서 사용할 수 있고 무료/유료로 구매하거나 다운로드 받을 수 있는 음성 엔진의 더욱 더 많은 정보를 찾으신다면, Extra voices 페이지를 보세요.

14.1. eSpeak NG

eSpeak NG 엔진은 NVDA에 직접적으로 내장되어 있고 특별한 드라이버나 구성요소를 설치할 필요가 없습니다. NVDA는 디폴트로 eSpeak NG 엔진을 사용하여 시작합니다. 이 엔진은 NVDA에 내장되어 있기 때문에 다른 시스템의 USB 메모리에서 NVDA를 실행하는 것은 매우 훌륭한 선택입니다.

eSpeak NG의 각 음성은 다양한 언어로 말합니다. eSpeak NG가 지원하는 다양한 언어들은 43개국 언어가 넘습니다.

음성의 소리를 대체하기 위하여 선택할 수 있는 다양한 variants가 있습니다.

14.2. Microsoft Speech API version 4 (SAPI 4)

SAPI 4는 마이크로소프트의 오래된 소프트웨어 음성엔진용 표준 규격입니다. 이 표준 규격을 준수하는 많은 음성 엔진들을 여러 회사와 웹사이트에서 구입하거나 무료로 다운받을 수 있습니다. NVDA에서 SAPI 4 음성 엔진을 이용할 때, (보이스 설정 대화상자 혹은 음성 엔진 설정 Ring)으로 액세스된) 사용가능한 음성들은 시스템에 설치된 모든 SAPI 4 엔진을 포함합니다.

만약 SAPI 4 음성 엔진을 설치하였지만 엔진이 NVDA의 음성 엔진 리스트에 표시되지 않는다면, https://activex.microsoft.com/activex/controls/sapi/spchapi.exe에서 SAPI 4.0 runtime binaries를 설치하세요.

14.3. Microsoft Speech API version 5 (SAPI 5)

SAPI 5는 마이크로소프트의 소프트웨어 음성 엔진용 표준 규격입니다. 시스템에 최소한 하나의 SAPI 5 음성엔진이 프리인스톨되어 있다 하더라도 이 표준 규격을 준수하는 많은 음성 엔진들을 여러 회사와 웹사이트에서 구입하거나 무료로 다운받을 수 있습니다. NVDA에서 SAPI 4 음성 엔진을 이용할 때, (보이스 설정 대화상자 혹은 음성 엔진 설정 Ring)으로 액세스된) 사용가능한 음성들은 시스템에 설치된 모든 SAPI 5 엔진을 포함합니다.

14.4. Microsoft Speech Platform

Microsoft Speech Platform은 보통 서버 기반의 음성 애플리케이션 개발에 이용되는 다양한 언어의 음성들을 제공합니다. 이 음성들은 NVDA에서도 사용할 수 있습니다.

이 음성들을 사용하려면 이하의 구성요소 두 가지를 설치해야 합니다.

  • Microsoft Speech Platform - Runtime (Version 11) , x86: https://www.microsoft.com/download/en/details.aspx?id=27225
  • Microsoft Speech Platform - Runtime Languages (Version 11): https://www.microsoft.com/download/en/details.aspx?id=27224
    • 이 페이지는 음성 인식 및 text-to-speech용 많은 파일들을 포함합니다. 희망하는 언어/음성용 TTS 데이터를 포함하는 파일을 선택하세요. 예를 들면, MSSpeech_TTS_en-US_ZiraPro.msi 파일은 미국식 영어 음성입니다.

14.5. Windows OneCore

Windows 10 이상 사용 시 OneCore(또는 Mobile)라는 음성 엔진을 사용할 수 있습니다. 여러 언어를 지원하고 각 언어마다 다른 보이스를 제공하며 마이크로소프트 SAPI 5 엔진보다 반응 속도가 빠릅니다. 윈도우 10 이상 사용시 Windows OneCore 엔진이 기본값으로 되어 있습니다(이전 윈도우 버전에서는 eSpeak NG 사용).

각 언어별 보이스 목록과 설치 방법에 대해서는 다음 마이크로소프트 웹사이트를 참조하세요: https://support.microsoft.com/en-us/help/22797/windows-10-narrator-tts-voices

16.1. 실행 명령

NVDA 시작시 하나 이상의 실행 명령 옵션을 통해 NVDA 실행 방식을 조절할 수 있습니다. 바탕 화면에 있는 NVDA 바로가기 아이콘(속성 페이지), 실행 대화상자(시작 메뉴/실행 또는 Windows+R) 또는 명령 Prompt에서 설정이 가능하며 이때 다수의 옵션을 입력할 수 있습니다. 실행 명령을 입력시 NVDA 실행 파일명과 각 옵션간 스페이스를 두어 입력하시기 바랍니다. 예를 들어 NVDA 설치중 바탕 화면 아이콘을 생성할때 -r 옵션이 추가되는데 이는 이전에 실행되던 NVDA를 종료시킨 후 재시작하라는 명령입니다. 다른 예로 --disable-addons 옵션인데 이는 현재 사용중인 추가 기능들을 일시 정지시키라는 명령이며 현재 발생한 문제가 추가 기능에 의한 문제인지를 파악하고자 할때 유용합니다.

한 예로 현재 실행중인 NVDA를 종료하려면 실행 대화상자에서 다음과 같이 입력합니다:

nvda -q

모든 실행 명령은 단어 명령이며 특정 옵션에 한해 약자 명령이 존재합니다. 다수의 약자 명령을 입력하려면 다음과 같이 입력하시면 됩니다:

nvda -mc 시작음 재생 불가 및 특정 환경 파일에 저장된 환경을 적용합니다.
nvda -mc --disable-addons 위와 같으며 또한 추가 기능을 일시 정지시킵니다.

여러 옵션은 세부 옵션을 지정할 수 있습니다(예: 실행 로그 레벨, NVDA 설정 파일 경로 지정 등). 세부 설정 입력시 약자 명령과 이에 대한 세부 명령 사이에 스페이스를 삽입하고 단어 명령과 이에 대한 세부 명령 사이에는 등호 기호(=)를 삽입합니다. 예를 들어:

nvda -l 10 실행 기록 레벨을 디버그로 설정합니다.
nvda --log-file=c:\nvda.log c:\nvda.log를 로그 파일로 지정합니다.
nvda --log-level=20 -f c:\nvda.log 실행 기록 레벨을 실행 정보로 설정하고 로그 내용을 c:\nvda.log 파일에 저장하도록 설정합니다.

  • 64비트 시스템: "HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\nvda"

본 레지스트리에서 변경할 수 있는 값들은 다음과 같습니다:

이름 타이프 설정값 설명
configInLocalAppData DWORD 0 (default)/disable, 1 (enable) 1로 변경 시 roaming application data가 아닌 local application data 폴더에 NVDA 설정이 저장됨.
serviceDebug DWORD 0 (default)/disable, 1 (enable) 1로 변경 시 보안 화면에 진입시 보안 모드로 전환하지 않음(파이썬 컨솔과 로그 뷰어를 사용할 수 있으나 여러 보안 문제가 발생할 수 있음으로 사용 권장 않 함).
